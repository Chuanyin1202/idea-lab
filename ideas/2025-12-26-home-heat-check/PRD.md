# 產品名稱
Home Heat Check

# One-line Pitch
將您的手機變成熱成像相機，幫助您找出家中熱量流失的地方，並提供簡單的修復建議。

# Background & Problem Statement
在寒冷的季節，許多家庭面臨著取暖費用上升的問題，尤其是在家庭聚會期間，卻發現家中某些區域仍然寒冷。這通常是因為家中存在無法察覺的熱量流失點，如窗戶、門和隔熱不良的牆壁。這些問題不僅影響舒適度，還導致不必要的高額能源費用。Home Heat Check 旨在解決這一問題，幫助用戶精確定位熱量流失的區域，並提供實用的修復建議。

# Target Users & Personas
- **家庭主婦/主夫**：關心家庭舒適度及能源費用，喜歡DIY修繕。
- **環保意識強的消費者**：希望降低能源消耗，減少碳足跡。
- **房屋擁有者**：對家居維護有興趣，尋求改善家居環境的解決方案。

# User Pain Points
- 無法精確定位家中熱量流失的具體位置。
- 高額的取暖費用，卻無法找到有效的解決方案。
- 對於修繕和改善措施缺乏清晰的指導和支持。

# Value Proposition
Home Heat Check 提供一個簡單易用的應用程序，通過熱成像技術幫助用戶發現家中熱量流失的具體位置，並提供針對性的修繕建議，讓用戶能夠輕鬆解決問題，降低能源費用，提升居家舒適度。

# Core Use Cases & User Stories
1. **發現熱量流失**：用戶使用手機掃描牆壁，應用程序顯示熱量流失的區域（藍色和紅色標記）。
2. **獲取修繕建議**：用戶點擊熱量流失的區域，獲得具體的修繕建議和DIY指導。
3. **聯繫承包商**：用戶可以輕鬆聯繫當地承包商進行專業修繕。

# MVP Scope
## Must-have
- 熱成像功能，能夠顯示熱量流失的區域。
- DIY修繕建議和指導。
- 聯繫當地承包商的功能。

## Nice-to-have
- 歷史數據跟蹤，用戶可以查看過去的修繕效果。
- 季節性掃描提醒功能。

# User Flow Overview
1. 用戶下載並安裝 Home Heat Check 應用程序。
2. 用戶註冊並登錄帳戶。
3. 用戶使用手機掃描家中各個區域。
4. 應用程序顯示熱量流失的區域。
5. 用戶點擊區域獲取修繕建議或聯繫承包商。

# Basic System Architecture
- **前端**：React Native（跨平台移動應用）
- **後端**：Node.js + Express（API伺服器）
- **數據庫**：MongoDB（用戶數據和歷史記錄）
- **熱成像技術**：集成第三方熱成像API或SDK

# Monetization Hypothesis
- 每月訂閱費用 $9，提供增值服務如季節性掃描提醒和歷史數據跟蹤。
- 每個合格的承包商推薦可獲得 $75 的收入。

# Success Metrics
- 目標 50,000 名訂閱用戶。
- 每月 500 次承包商推薦。
- 用戶滿意度調查，目標 80% 以上滿意度。

# Risks & Assumptions
- 風險：市場競爭激烈，可能面臨其他類似應用的挑戰。
- 假設：用戶願意為節省能源費用和提升舒適度支付月費。
