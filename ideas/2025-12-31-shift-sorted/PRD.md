# 產品名稱
Shift Sorted

# One-line Pitch
自動填補服務業取消班次的排班軟體，讓管理者不再為人力資源的變動而煩惱。

# Background & Problem Statement
在服務業中，尤其是移動公司、清潔服務、活動人員和餐飲業，班次的排定往往面臨人員取消、缺席或忘記的問題。這使得管理者每週都要面對相同的困境：誰有空？誰具備所需的技能？誰離工作地點最近？現有的排班工具多數是針對固定班次的餐廳設計，無法滿足靈活變動的工作需求。Shift Sorted 將解決這一問題，通過自動匹配工作人員和班次，減少管理者的壓力。

# Target Users & Personas
- **移動公司經理**：需要靈活調配人員以應對突發情況。
- **清潔服務業主**：面對高流動性員工，需快速找到替代人員。
- **活動策劃者**：需即時調整人員配置以確保活動順利進行。
- **餐飲業者**：需要快速填補取消的班次。

# User Pain Points
- 班次取消時，難以迅速找到合適的替代人員。
- 現有工具無法支持靈活的排班需求。
- 管理者需花費大量時間在聯絡員工上。
- 無法即時掌握人員的可用性和技能。

# Value Proposition
Shift Sorted 提供一個自動化的排班解決方案，能夠即時匹配可用的工作人員，減少管理者的負擔，提升工作效率，並確保服務的連續性。透過簡單的操作，管理者可以專注於業務發展，而非人力資源的管理。

# Core Use Cases & User Stories
1. **班次發布**：管理者在系統中發布班次需求，指定所需技能和工作地點。
   - *作為一名經理，我希望能夠快速發布班次，以便能夠找到合適的工作人員。*
   
2. **自動匹配**：系統自動根據可用性和技能匹配合適的工作人員。
   - *作為一名經理，我希望系統能自動找到替代人員，這樣我就不必手動聯絡每個人。*

3. **班次聲明**：工作人員可以通過手機應用聲明班次。
   - *作為一名工作人員，我希望能夠隨時查看可用的班次並迅速聲明。*

# MVP Scope (Must-have vs Nice-to-have)
## Must-have
- 班次發布功能
- 自動匹配系統
- 工作人員手機應用
- 可用性和技能管理

## Nice-to-have
- 與 Google Calendar 和 Slack 的整合
- 實時通知和提醒功能
- 數據分析報告（如班次填補率）

# User Flow Overview
1. 管理者登入系統，發布班次需求。
2. 系統自動匹配可用的工作人員。
3. 工作人員收到通知，並在手機上查看班次。
4. 工作人員聲明班次，系統更新班次狀態。
5. 管理者隨時查看班次覆蓋情況。

# Basic System Architecture
- **前端**：React.js (Web 應用) / React Native (手機應用)
- **後端**：Node.js + Express
- **數據庫**：MongoDB
- **第三方整合**：Google Calendar API, Slack API
- **雲服務**：AWS 或 Azure

# Monetization Hypothesis
每位活躍工作人員每月收費 $30-50，透過訂閱模式進行收費。預計在首次防止錯過工作或緊急加班時，軟體將自我回本。

# Success Metrics
- 每月活躍用戶數
- 班次填補率
- 用戶滿意度調查結果
- 客戶留存率

# Risks & Assumptions
- **風險**：市場接受度不如預期，競爭對手的出現。
- **假設**：用戶願意為自動化排班支付訂閱費用，並且能夠接受新工具的使用。
