# 產品名稱
Shop The Room

# One-line Pitch
將房間照片轉換為可購買的家具清單，讓每個靈感瞬間成為現實。

# Background & Problem Statement
許多人經常在旅館大廳、Airbnb或社交媒體上截取靈感照片，然而，這些靈感往往隨著時間而消失在相簿中，未能轉化為實際的購買行為。現有的家具購物平台無法有效地將這些靈感轉化為具體的購物選擇，導致消費者失去興趣。Shop The Room 旨在解決這一問題，通過簡單的照片識別技術，將房間中的每一件家具轉化為可購買的清單。

# Target Users & Personas
- **家庭裝潢愛好者**：喜歡在家中進行裝潢和設計，經常在Pinterest上保存靈感。
- **年輕租客**：希望在有限的預算內美化租住空間，對價格敏感。
- **室內設計師**：尋找靈感並希望快速找到合適的家具供客戶選擇。

# User Pain Points
1. 難以找到與靈感照片中相符的家具。
2. 購物過程繁瑣，需要在多個網站上搜索。
3. 缺乏價格比較工具，無法找到最優惠的價格。

# Value Proposition
Shop The Room 提供一個簡單的解決方案，讓用戶只需拍攝或上傳房間照片，即可獲得該房間中所有家具的詳細資訊，包括價格、購買連結和替代品選擇。這不僅節省了用戶的時間，還能提高購買的成功率。

# Core Use Cases & User Stories
1. **用戶上傳照片**：用戶拍攝或上傳一張房間照片，應用程式識別出所有家具。
2. **查看結果**：用戶看到每件家具的詳細資訊和購買連結。
3. **比較價格**：用戶可以查看不同商家的價格，選擇最優惠的選項。
4. **收藏靈感**：用戶可以將喜歡的家具添加到收藏夾，方便日後查看。

# MVP Scope (Must-have vs Nice-to-have)
### Must-have
- 照片識別技術，能準確識別主要家具（沙發、桌子、燈具）。
- 整合主要家具電商平台的數據庫（如Wayfair、West Elm）。
- 提供價格比較功能。
- 基本的用戶界面和用戶體驗設計。

### Nice-to-have
- 提供替代品建議。
- 高級功能，如無限掃描、風格匹配和佈局建議的訂閱服務。
- 社交分享功能，讓用戶能分享自己的靈感。

# User Flow Overview
1. 用戶打開應用程式。
2. 用戶選擇拍攝新照片或從相簿中上傳。
3. 應用程式分析照片並顯示識別出的家具。
4. 用戶查看家具詳細資訊和購買連結。
5. 用戶可以選擇收藏或直接購買。

# Basic System Architecture
- **前端**：React Native（移動應用程式）
- **後端**：Node.js + Express
- **數據庫**：MongoDB
- **圖片識別技術**：使用TensorFlow或OpenCV
- **第三方API整合**：電商平台的API進行數據拉取

# Monetization Hypothesis
- 透過聯盟行銷獲取每次購買的佣金（5-15%）。
- 提供每月29美元的高級訂閱服務，包含無限掃描、風格匹配和佈局建議。

# Success Metrics
- 每月活躍用戶數（MAU）。
- 用戶轉化率（從掃描到購買的比例）。
- 每月訂閱用戶數。
- 用戶滿意度調查結果。

# Risks & Assumptions
- 風險：圖片識別技術的準確性可能不夠高，影響用戶體驗。
- 假設：用戶願意為高級功能支付額外費用，並且對於價格比較功能有需求。
