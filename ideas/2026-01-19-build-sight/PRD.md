# 產品名稱
Build Sight

# One-line Pitch
一個建築材料追蹤器，幫助承包商在供應商之前發現延遲。

# Background & Problem Statement
在建築行業中，材料的準時交付對於項目的進度至關重要。然而，供應商經常無法準時交貨，導致承包商面臨工期延誤和成本增加的風險。傳統上，承包商依賴供應商的承諾，但這些承諾往往不可靠，造成了項目延誤和客戶不滿。Build Sight 將通過實時追蹤材料狀態，讓承包商能夠提前識別潛在的延遲，從而有效管理項目進度。

# Target Users & Personas
- **小型承包商**：通常管理少量項目，對材料的依賴性高，尋求提高效率和減少延誤。
- **中型建築公司**：擁有多個工地，需協調多個供應商，對材料管理有更高的需求。
- **項目經理**：負責監控項目進度，尋求工具來簡化材料追蹤和預測。

# User Pain Points
1. 供應商的交貨時間不準確，導致工期延誤。
2. 難以實時獲取材料的庫存狀態和交付進度。
3. 手動追蹤材料狀態的效率低，容易出錯。
4. 缺乏有效的預測工具，無法提前識別潛在問題。

# Value Proposition
Build Sight 提供一個集中式的材料追蹤平台，通過自動化數據收集和分析，讓承包商能夠在問題發生之前識別延遲，並及時做出調整，從而提高項目效率，減少成本損失。

# Core Use Cases & User Stories
1. **材料狀態追蹤**：作為一名承包商，我希望能夠實時查看所有材料的庫存狀態，這樣我可以提前識別延遲。
2. **預測交貨時間**：作為一名項目經理，我希望系統能根據歷史數據預測材料的交貨時間，這樣我可以更好地安排工人和資源。
3. **延遲警報**：作為一名小型承包商，我希望在材料延遲時收到即時通知，以便我能夠迅速做出應對措施。

# MVP Scope (Must-have vs Nice-to-have)
**Must-have:**
- 實時庫存追蹤
- 延遲警報系統
- 用戶友好的儀表板
- 數據分析和報告功能

**Nice-to-have:**
- 自動化供應商聯繫功能
- 多用戶協作功能
- 整合其他建築管理工具的API

# User Flow Overview
1. 用戶登錄系統。
2. 用戶輸入材料訂單信息。
3. 系統自動拉取供應商的庫存數據。
4. 系統分析數據並生成報告。
5. 用戶查看儀表板，接收延遲警報。
6. 用戶根據報告做出調整。

# Basic System Architecture
- **前端**：React.js
- **後端**：Node.js + Express
- **數據庫**：MongoDB
- **數據抓取**：使用 Python 的爬蟲技術從供應商網站抓取庫存數據
- **雲服務**：AWS 或 Google Cloud

# Monetization Hypothesis
- 單個承包商收費 $200-500/月
- 多個工地的公司收費 $500-1,000/月
- 提供增值服務如數據分析報告和專業諮詢服務

# Success Metrics
- 用戶增長率
- 每月活躍用戶數
- 用戶滿意度調查結果
- 減少的項目延遲時間

# Risks & Assumptions
- **風險**：供應商數據的準確性和可用性可能影響系統的有效性。
- **假設**：承包商願意支付額外費用以獲得更高的效率和減少延遲。
