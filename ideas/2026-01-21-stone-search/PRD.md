# 產品名稱
Stone Search

# One-line Pitch
將墓碑數字化，讓家庭能快速找到祖先的安息之地。

# Background & Problem Statement
隨著時間的推移，許多家庭面臨著尋找已故親人的困難，特別是當墓碑的資料未被數字化時。許多墓地的記錄仍然是紙本或未經整理的，導致後代在尋找家族歷史時遇到障礙。尤其是對於像瑪格麗特這樣的年長者，她的記憶逐漸模糊，無法準確告訴後代親人的埋葬地點。這種情況不僅影響家庭的情感連結，也使得墓地管理者面臨數據管理的挑戰。

# Target Users & Personas
1. **家庭成員**：如瑪格麗特，年長者，對家族歷史有濃厚興趣，尋找已故親人的安息之地。
2. **墓地管理者**：負責管理墓地的工作人員，面臨資料整理與數字化的需求。
3. **家譜愛好者**：熱衷於研究家族歷史的個人，願意為尋找資料付費。

# User Pain Points
- 家庭成員無法找到已故親人的埋葬地點。
- 墓地管理者缺乏數字化工具來管理和查詢墓碑資料。
- 家譜愛好者面對繁瑣的資料搜尋過程。

# Value Proposition
Stone Search 提供一個簡單易用的平台，將墓碑資料數字化，並透過 AI 技術快速提取並整理資料，讓家庭能夠輕鬆找到親人的安息之地，同時幫助墓地管理者提升管理效率。

# Core Use Cases & User Stories
1. **家庭成員的使用場景**：
   - 作為一名家庭成員，我想上傳墓碑照片，讓 AI 幫我提取資料，這樣我可以快速找到親人的埋葬地點。
   
2. **墓地管理者的使用場景**：
   - 作為一名墓地管理者，我希望能將所有墓碑資料數字化，這樣我可以更有效地管理資料並提供給家庭查詢。

3. **家譜愛好者的使用場景**：
   - 作為一名家譜愛好者，我希望能透過平台搜索祖先的墓碑資料，這樣我可以更方便地研究家族歷史。

# MVP Scope (Must-have vs Nice-to-have)
## Must-have
- 基本的照片上傳功能
- AI 文字識別技術來提取墓碑資料（姓名、出生日期、死亡日期）
- 數據庫建立與基本搜尋功能（依姓名、日期、地點搜尋）
- 初步的用戶界面設計

## Nice-to-have
- GPS 標記功能以標示精確位置
- 社群功能讓用戶分享與交流
- 高級搜尋過濾器（如根據關係搜尋）

# User Flow Overview
1. 用戶註冊並登入平台。
2. 用戶上傳墓碑照片。
3. 系統使用 AI 技術提取資料並儲存至數據庫。
4. 用戶可以透過搜尋功能查找資料。
5. 用戶可以查看墓碑的具體位置和相關信息。

# Basic System Architecture
- **前端**：React.js
- **後端**：Node.js + Express
- **數據庫**：MongoDB
- **AI 技術**：Google Cloud Vision API 或 Amazon Rekognition
- **雲存儲**：AWS S3

# Monetization Hypothesis
- 家庭用戶每月收取 $10-$30 的訂閱費用，提供高級搜尋功能及資料存取。
- 墓地管理者每年收取 $500-$2000 的服務費用，提供數字化與管理工具。

# Success Metrics
- 每月活躍用戶數（MAU）
- 用戶轉換率（從免費試用轉為付費用戶）
- 每個墓地的資料數量與完整性
- 用戶滿意度調查結果

# Risks & Assumptions
- **風險**：AI 文字識別的準確性可能受限於墓碑的損壞程度。
- **假設**：家庭用戶和墓地管理者都願意為數字化服務付費，並且市場需求持續增長。
