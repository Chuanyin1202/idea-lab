# 產品名稱
One Property

# One-line Pitch
簡單的稅務工具，幫助意外房東輕鬆管理租金收入和報稅。

# Background & Problem Statement
隨著房地產市場的變化，越來越多的人成為意外房東，無論是繼承房產還是將空閒空間出租以補貼房貸。這些新手房東面臨著複雜的稅務問題，尤其是在報稅季節。他們通常缺乏相關知識，容易在稅務上出錯，導致潛在的罰款和損失。現有的稅務工具大多針對專業投資者，無法滿足這些意外房東的需求。

# Target Users & Personas
- **意外房東**：例如繼承房產的年輕人，對房地產管理和稅務知識了解有限。
- **房屋共享者**：例如將空閒房間出租的年輕專業人士，尋求簡單的租金管理工具。
- **個人理財愛好者**：活躍於 Reddit 等社交平台，尋找簡單有效的稅務解決方案。

# User Pain Points
1. 缺乏稅務知識，無法正確分類和報告支出。
2. 難以追蹤和管理與租金相關的費用。
3. 對於稅務合規性有疑慮，擔心罰款和法律問題。
4. 希望能夠快速簡便地生成報稅所需的文件。

# Value Proposition
One Property 提供一個簡單易用的稅務管理工具，專為意外房東設計。用戶只需連接銀行帳戶，並通過手機拍攝收據，AI 系統將自動將費用分類，並在年末生成所需的稅務文件，幫助用戶輕鬆報稅，避免稅務風險。

# Core Use Cases & User Stories
1. **用戶連接銀行帳戶**：作為一名用戶，我希望能夠連接我的銀行帳戶，以便自動導入交易記錄。
2. **拍攝收據**：作為一名用戶，我希望能夠拍攝收據，並自動將其分類到正確的費用類別中。
3. **生成稅務文件**：作為一名用戶，我希望在年末能夠生成清晰的 Schedule E 和折舊報告，方便我報稅。

# MVP Scope
- **Must-have**:
  - 銀行帳戶連接功能
  - 收據拍攝和自動分類功能
  - 生成 Schedule E 和折舊報告的功能
- **Nice-to-have**:
  - 提供稅務諮詢服務的選項
  - 整合常見稅務問題的問答庫
  - 用戶社群功能，讓用戶分享經驗

# User Flow Overview
1. 用戶註冊並連接銀行帳戶。
2. 用戶拍攝收據並上傳。
3. 系統自動分類費用並顯示在用戶面板上。
4. 到報稅季節，用戶生成所需的稅務文件。

# Basic System Architecture
- **前端**: React Native (行動應用)
- **後端**: Node.js + Express
- **數據庫**: MongoDB
- **第三方整合**: Plaid (銀行帳戶連接), OCR API (收據識別)

# Monetization Hypothesis
- 核心產品訂閱費用：每月 $19-29。
- 稅季期間提供 CPA 諮詢服務，收費 $75。

# Success Metrics
- 每月活躍用戶數 (MAU)
- 用戶留存率
- 轉換率（從免費試用到付費訂閱）
- 用戶滿意度調查結果

# Risks & Assumptions
- **風險**: 用戶對於數據安全的擔憂可能會影響他們的使用意願。
- **假設**: 用戶對於簡化稅務管理的需求將持續增長，並願意為此支付訂閱費用。
