# 產品名稱
Sympatico

## One-line Pitch
一個超越背景檢查的護理人員與客戶匹配平台，專注於個性與照護風格的兼容性。

## Background & Problem Statement
隨著人口老齡化，對於在家護理服務的需求急劇上升。然而，現有的護理人員與客戶匹配系統往往僅依賴於背景檢查，未能考慮到個性和照護風格的兼容性，導致護理人員與客戶之間的配對失敗率高達65%。這不僅影響了客戶的滿意度，也增加了護理機構的成本，因為每一次失敗的配對都可能造成高達$2,500的損失。

## Target Users & Personas
- **護理機構經營者**：希望提高護理人員與客戶的匹配成功率，減少流失率。
- **護理人員**：尋找能夠與自己風格相符的客戶，以提高工作滿意度。
- **家庭照護者**：希望找到合適的護理人員，以確保家中老人的照護質量。

## User Pain Points
- 護理人員與客戶的匹配僅依賴於表面條件，缺乏深度了解。
- 高流失率導致護理機構需不斷招聘和培訓新員工，增加成本。
- 家庭對於護理人員的選擇感到困惑，不知道如何評估兼容性。

## Value Proposition
Sympatico 提供一個基於個性和照護風格的匹配系統，通過自動化的數據分析，幫助護理機構找到最合適的護理人員，從而提高配對成功率，降低流失率，並最終提升客戶滿意度。

## Core Use Cases & User Stories
1. **護理機構經營者**：希望在系統中查看護理人員的兼容性評分，以便做出更明智的雇用決策。
2. **護理人員**：希望填寫個性和照護風格的問卷，以便找到最合適的客戶。
3. **家庭照護者**：希望查看護理人員的兼容性報告，以便選擇最適合的護理人員。

## MVP Scope
### Must-have
- 與 ClearCare API 整合，獲取護理人員的可用性和客戶記錄。
- 兩個10題的問卷，分別針對護理人員和家庭照護者。
- 兼容性評分算法，根據問卷結果生成匹配建議。
- 在現有工作流程中顯示匹配結果。

### Nice-to-have
- 數據分析儀表板，顯示匹配成功率和流失率的趨勢。
- 客戶和護理人員的反饋機制，以持續改進匹配算法。

## User Flow Overview
1. 護理機構經營者登入系統。
2. 系統自動從 ClearCare 獲取護理人員和客戶的數據。
3. 護理人員和家庭照護者分別填寫問卷。
4. 系統根據問卷結果計算兼容性評分。
5. 經營者查看匹配結果並進行安排。

## Basic System Architecture
- **前端**：React.js
- **後端**：Node.js + Express
- **數據庫**：MongoDB
- **API 整合**：ClearCare API
- **數據分析**：Python + Pandas

## Monetization Hypothesis
- 小型護理機構每月收費 $29-199，大型機構每月收費 $1,000-2,000，根據使用的功能和數據分析服務的深度進行定價。

## Success Metrics
- 每月活躍用戶數 (MAU)
- 護理人員與客戶的匹配成功率
- 客戶滿意度調查結果
- 流失率降低百分比

## Risks & Assumptions
- **風險**：護理機構對新系統的接受度不高，可能影響初期用戶增長。
- **假設**：護理人員和家庭照護者能夠準確填寫問卷，並真實反映其需求和期望。
