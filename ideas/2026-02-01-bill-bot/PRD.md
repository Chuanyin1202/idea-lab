# 產品名稱
Bill Bot

## One-line Pitch
為患者提供醫療帳單談判服務，幫助他們在面對高額醫療費用時獲得合理的價格。

## Background & Problem Statement
在美國，許多患者面對的醫療帳單往往高得令人震驚，醫療保險公司僅支付每美元22美分，患者卻需支付全額。這導致許多人在面對高額帳單時感到無助與焦慮，尤其是在手術後或重大診斷後。患者往往不知道自己可以進行談判，或是缺乏精力去面對這場艱難的戰鬥。這樣的情況使得醫療機構得以利用患者的無知和疲憊，收取不合理的費用。

## Target Users & Personas
- **患者**：面對高額醫療帳單的患者，尤其是那些剛經歷手術或重大健康問題的人。
- **社會工作者**：經常與患者接觸，能夠提供支持和指導的專業人士。
- **患者倡導者**：專注於幫助患者獲得合理醫療服務的組織或個人。

## User Pain Points
- 對醫療帳單的複雜性缺乏了解。
- 缺乏時間和精力去進行帳單談判。
- 對於談判過程的恐懼和不安。
- 不知道如何尋找幫助或資源。

## Value Proposition
Bill Bot 將為患者提供一個簡單、有效的醫療帳單談判工具，讓他們能夠在不需提前支付的情況下，獲得合理的醫療費用。透過專業的談判團隊和數據分析，患者將能夠輕鬆地減少醫療費用，減輕經濟負擔。

## Core Use Cases & User Stories
1. **使用者上傳醫療帳單**：患者將醫療帳單以PDF或照片形式上傳至平台。
2. **系統分析帳單**：Bill Bot 使用OCR技術提取帳單項目，並與CMS基準進行比對，標記可談判的費用。
3. **進行談判**：系統自動與醫療機構的帳單部門進行談判，並記錄談判過程及結果。
4. **獲得節省金額**：患者在談判成功後，根據節省金額支付20%的服務費。

## MVP Scope (Must-have vs Nice-to-have)
### Must-have
- 上傳醫療帳單的功能。
- OCR技術提取帳單項目。
- 與醫療機構進行談判的基本框架。
- 記錄談判結果和數據分析功能。

### Nice-to-have
- 使用者界面優化和多語言支持。
- 數據儀表板，顯示談判成功率和節省金額統計。
- 社交媒體整合，方便患者分享經驗。

## User Flow Overview
1. 使用者註冊並登錄。
2. 上傳醫療帳單。
3. 系統分析並標記可談判項目。
4. Bill Bot 自動開始談判。
5. 使用者接收談判結果及節省金額通知。

## Basic System Architecture
- **前端**：React.js
- **後端**：Node.js + Express
- **數據庫**：MongoDB
- **OCR技術**：Tesseract.js
- **雲存儲**：AWS S3
- **API整合**：與醫療機構的API進行數據交互。

## Monetization Hypothesis
Bill Bot 將根據每次成功談判所節省的金額收取20%的服務費。這將吸引患者使用該服務，因為他們在支付前不需要任何 upfront 費用。

## Success Metrics
- 每月使用者增長率。
- 成功談判的帳單百分比。
- 平均每次談判節省的金額。
- 使用者滿意度評分。

## Risks & Assumptions
- **風險**：醫療機構可能不願意與自動化系統進行談判，導致談判成功率低。
- **假設**：患者對於使用數位工具進行醫療帳單談判持開放態度，並願意分享其經驗以促進服務的擴展。
