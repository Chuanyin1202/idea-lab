# 產品名稱
Benefits Found

# One-line Pitch
幫助年長者輕鬆找到他們應得的福利，讓未來的生活更有保障。

# Background & Problem Statement
隨著年齡的增長，許多年長者面臨著複雜的福利申請流程，這些流程通常分散在不同的機構和網站上。許多年長者並不知道他們有資格獲得哪些福利，這導致大量的資源未被充分利用。政府的福利資訊雖然公開，但缺乏整合，讓年長者難以獲得所需的幫助。

# Target Users & Personas
- **年長者**：65歲以上，可能是單身或有伴侶，對於福利申請流程感到困惑。
- **照顧者**：為多位年長者提供照顧的專業人員，需協助他們獲得應有的福利。
- **家庭成員**：關心年長者福利的子女或親屬，想要幫助他們獲得資源。

# User Pain Points
1. 難以找到適合自己的福利資訊。
2. 申請流程繁瑣，缺乏指導。
3. 資源未被充分利用，導致經濟壓力。
4. 不同機構的資訊不一致，容易造成混淆。

# Value Proposition
Benefits Found 提供一個簡單易用的平台，讓年長者和照顧者能夠快速找到和申請他們有資格獲得的福利，從而減少經濟壓力，提升生活質量。

# Core Use Cases & User Stories
1. **年長者使用者故事**：作為一位72歲的寡婦，我希望能夠輸入我的基本資訊，快速找到我可以申請的福利，以便改善我的經濟狀況。
2. **照顧者使用者故事**：作為一位照顧者，我希望能夠為多位客戶進行福利搜尋，並追蹤他們的申請進度，以便提供更好的服務。

# MVP Scope (Must-have vs Nice-to-have)
- **Must-have**:
  - 基本的用戶註冊和登錄功能。
  - 10個字段的基本資料輸入表單（年齡、收入、退伍軍人狀態、婚姻狀況、房屋擁有權、郵遞區號）。
  - 與政府福利數據庫的整合（例如Benefits.gov API）。
  - 自動生成的福利匹配結果。

- **Nice-to-have**:
  - 申請進度追蹤功能。
  - 用戶反饋和評價系統。
  - 多語言支持。

# User Flow Overview
1. 用戶註冊或登錄。
2. 填寫基本資料表單。
3. 系統根據輸入的資料查詢相應的福利。
4. 顯示匹配的福利資訊及申請指導。
5. 用戶可選擇進一步追蹤申請進度。

# Basic System Architecture
- **前端**：React.js
- **後端**：Node.js + Express
- **數據庫**：Supabase
- **API整合**：Benefits.gov API
- **雲服務**：AWS或Google Cloud

# Monetization Hypothesis
- 個人用戶免費使用基本功能，提供高級訂閱服務（$49/月）以解鎖無限制搜尋和申請追蹤功能。
- 照顧機構可訂閱專業版（$300/月），提供自動檢查新客戶的功能。

# Success Metrics
1. 每月活躍用戶數（MAU）。
2. 用戶完成福利申請的比例。
3. 用戶滿意度評分。
4. 訂閱轉換率。

# Risks & Assumptions
- **風險**：用戶對於數據隱私的擔憂可能影響註冊率。
- **假設**：年長者及其照顧者願意使用數位工具來尋找福利資訊。
