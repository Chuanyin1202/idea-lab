# 產品名稱
Speech Play

# One-line Pitch
將語言治療變成遊戲，讓孩子在家中輕鬆練習，家長和治療師都能隨時追蹤進度。

# Background & Problem Statement
許多語言治療師在每週的治療後，會將練習工作表帶回家，但大多數孩子在家中無法持續練習，導致治療效果不佳。傳統的紙本作業容易被遺忘或忽略，造成家長和孩子的挫敗感。Speech Play旨在透過遊戲化的方式，提升孩子的參與度，並簡化家長和治療師的追蹤過程。

# Target Users & Personas
1. **孩子**：年齡在3-12歲之間，正在接受語言治療的兒童。
2. **家長**：希望能在家中支持孩子練習的父母，對科技產品持開放態度。
3. **語言治療師**：專業的治療師，尋求有效的工具來提高治療效率和追蹤進度。

# User Pain Points
- 孩子對傳統作業缺乏興趣，無法持續練習。
- 家長難以記住治療師的指示，無法有效協助孩子。
- 治療師無法準確追蹤孩子的練習進度和困難點。

# Value Proposition
Speech Play透過遊戲化的練習方式，讓孩子在玩樂中學習，並提供家長和治療師即時的進度追蹤，減少溝通成本，提升治療效果。

# Core Use Cases & User Stories
1. **孩子使用者故事**：作為一個孩子，我希望能夠在手機上玩有趣的語言練習遊戲，這樣我就能在家中輕鬆練習，而不會感到無聊。
2. **家長使用者故事**：作為一位家長，我希望能隨時查看孩子的練習進度，這樣我就能夠有效地支持他們的學習。
3. **治療師使用者故事**：作為一位語言治療師，我希望能夠查看孩子的練習數據，這樣我就能在下次會面時提供更具針對性的建議。

# MVP Scope (Must-have vs Nice-to-have)
## Must-have
- 基本的遊戲化練習模組（15-20個遊戲）
- 家長進度追蹤介面（進度條、歷史記錄）
- 治療師儀表板（遊戲分配、完成率、停滯點）

## Nice-to-have
- 社交分享功能（讓孩子分享成就）
- 語音識別反饋系統（即時反饋孩子的發音）
- 個性化推薦系統（根據孩子的進度推薦適合的遊戲）

# User Flow Overview
1. 治療師在系統中分配遊戲給孩子。
2. 家長下載並登入應用程式。
3. 孩子在應用程式中選擇遊戲並開始練習。
4. 系統記錄練習時間和進度。
5. 家長和治療師可以查看進度報告。

# Basic System Architecture
- **前端**：使用 React Native 開發跨平台應用。
- **後端**：使用 Firebase 存儲會話數據和用戶資料。
- **分析工具**：整合 Google Analytics 以追蹤用戶行為。

# Monetization Hypothesis
- 家庭用戶每月支付 $20，獲得完整遊戲庫和進度追蹤功能。
- 診所用戶每月支付 $50，享有多病人管理和分析功能。
- 提供免費基礎版以吸引用戶。

# Success Metrics
- 每月活躍用戶數（MAU）
- 用戶留存率
- 家長和治療師的滿意度調查
- 每月收入（MRR）

# Risks & Assumptions
- 風險：市場對遊戲化學習的接受度不高，可能影響用戶增長。
- 假設：家長和治療師會願意支付訂閱費用以獲得更好的治療效果。
