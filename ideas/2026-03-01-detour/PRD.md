# 產品名稱
Detour

# One-line Pitch
一款專為千禧世代設計的微退休模擬器，幫助用戶計算職業生涯中的工作與休息對長期財富的影響。

# Background & Problem Statement
隨著職業生涯的多樣化，許多千禧世代面臨著不同於父母的職業規劃。他們可能會選擇在職業生涯中休息、旅行或啟動副業。然而，現有的退休計算器無法有效模擬這種非傳統的職業路徑，導致年輕人無法準確評估其長期財務影響。Detour 旨在填補這一空白，提供一個能夠考量工作與休息時間的財務計劃工具。

# Target Users & Personas
- **主要用戶**: 32歲的軟體工程師，計劃在職業生涯中休息一年去東南亞旅行。
- **其他用戶**: 其他千禧世代專業人士，擁有不規則的職業路徑，對財務計劃有需求。

# User Pain Points
1. 現有的退休計算器無法考量工作與休息的交替。
2. 缺乏針對非傳統職業路徑的財務模擬工具。
3. 用戶對未來財務狀況的預測不準確，缺乏信心。

# Value Proposition
Detour 提供一個專為千禧世代設計的微退休模擬器，能夠精確計算不同工作與休息模式對長期財富的影響，幫助用戶做出明智的財務決策。

# Core Use Cases & User Stories
1. **用戶故事**: 作為一名軟體工程師，我希望能夠輸入我的收入和預計的休息時間，以便計算出退休時的淨資產。
2. **用戶故事**: 作為一名千禧世代專業人士，我希望能夠連接我的銀行帳戶，以便獲取真實的財務數據，而不是依賴估算。

# MVP Scope (Must-have vs Nice-to-have)
- **Must-have**:
  - 基本的網頁計算器，能夠輸入收入、儲蓄和休息時間。
  - 顯示不同情境下的預測淨資產。
  - 連接 Plaid 獲取真實帳戶數據。
  
- **Nice-to-have**:
  - 進階的目標追蹤功能。
  - 市場變動警報系統。
  - 社交分享功能，讓用戶分享模擬結果。

# User Flow Overview
1. 用戶訪問網站並註冊帳戶。
2. 輸入個人財務數據（收入、儲蓄、計劃的休息時間）。
3. 系統計算並顯示不同情境下的預測淨資產。
4. 用戶可選擇連接銀行帳戶以獲取真實數據。
5. 用戶可選擇分享結果至社交媒體。

# Basic System Architecture
- **前端**: React.js
- **後端**: Node.js + Express
- **數據庫**: MongoDB
- **第三方服務**: Plaid API（用於銀行數據連接）
- **雲服務**: AWS（用於部署和存儲）

# Monetization Hypothesis
- 提供免費的基本計算器吸引用戶，進而推廣 $19/月的高級訂閱服務，解鎖進階功能。
- 針對企業客戶，提供年費授權的B2B產品，幫助HR團隊為員工提供財務規劃工具。

# Success Metrics
1. 用戶註冊數量。
2. 每月活躍用戶數。
3. 高級訂閱轉化率。
4. 用戶滿意度和反饋。

# Risks & Assumptions
- **風險**: 用戶對於提供銀行數據的隱私擔憂可能影響使用率。
- **假設**: 千禧世代對於非傳統職業路徑的財務規劃需求將持續增長。
