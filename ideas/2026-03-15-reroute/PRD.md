# 產品名稱
Reroute

## One-line Pitch
自動化航班延誤賠償及重訂服務，讓頻繁旅行者無需再為航班問題而煩惱。

## Background & Problem Statement
航空公司在延誤或取消航班時，根據法律規定需要支付賠償。然而，許多旅客在首次申請時遭到拒絕，航空公司依賴於旅客不會進一步跟進的心理。這使得旅客不僅面臨損失，還要花費大量時間在電話上等待客服。Reroute 將透過即時監控航班狀態，自動化處理賠償申請及重訂流程，從而減少旅客的麻煩。

## Target Users & Personas
1. **頻繁商務旅行者**：需要快速解決航班問題，以便專注於工作。
2. **休閒旅客**：希望簡化旅行過程，減少因航班問題而造成的焦慮。
3. **企業旅行經理**：需要有效管理公司的旅行支出，並確保員工能夠順利出行。

## User Pain Points
- 航空公司賠償申請流程繁瑣且耗時。
- 航班延誤或取消時，重訂流程不夠便捷。
- 缺乏即時的航班狀態更新和賠償資格通知。
- 對於賠償申請的成功率缺乏透明度。

## Value Proposition
Reroute 將提供一個自動化的平台，實時監控航班狀態，並在延誤或取消發生時自動處理賠償申請及重訂，讓用戶省去繁瑣的手續和等待時間，並提高賠償成功率。

## Core Use Cases & User Stories
1. **航班延誤自動重訂**：
   - 當航班延誤時，系統自動根據用戶的偏好進行重訂。
   - 用戶收到即時通知，告知重訂結果。

2. **自動化賠償申請**：
   - 當航班取消時，系統自動檢查賠償資格並提交申請。
   - 用戶在航班延誤結束前收到賠償申請的進度更新。

## MVP Scope (Must-have vs Nice-to-have)
### Must-have
- 航班實時監控系統。
- 自動化賠償申請生成與提交功能。
- 用戶偏好設置及重訂邏輯。
- 基本的用戶通知系統。

### Nice-to-have
- 賠償成功率分析與報告功能。
- 企業用戶管理界面及報告工具。
- 整合多家航空公司的賠償規則。

## User Flow Overview
1. 用戶註冊並設置個人偏好（座位類型、航空公司等）。
2. 系統實時監控用戶的航班狀態。
3. 當航班延誤或取消時，系統自動重訂並提交賠償申請。
4. 用戶收到即時通知，並可查看申請進度。

## Basic System Architecture
- **前端**：React.js
- **後端**：Node.js + Express
- **數據庫**：MongoDB
- **實時監控**：使用第三方航空數據API
- **通知系統**：Twilio API（用於短信通知）

## Monetization Hypothesis
- 個人用戶每月收費 $29。
- 企業用戶每位用戶收費 $10-$30，並對成功賠償收取 25% 的佣金。

## Success Metrics
- 每月活躍用戶數（MAU）。
- 賠償申請成功率。
- 用戶滿意度調查結果。
- 企業用戶的續訂率。

## Risks & Assumptions
- **風險**：航空公司可能會改變賠償政策，影響業務模式。
- **假設**：用戶願意為自動化服務支付月費，並且對賠償申請的成功率有足夠的信心。
