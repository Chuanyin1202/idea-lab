# 產品名稱
CryptoLegacy

# One-line Pitch
為非技術性加密貨幣持有者提供安全、簡單的加密資產繼承解決方案。

# Background & Problem Statement
隨著加密貨幣的普及，越來越多的人開始持有數位資產。然而，許多持有者並不具備技術背景，導致他們在資產繼承方面面臨困難。根據統計，超過1400億美元的加密貨幣因密碼遺失或持有者去世而無法取回。現有的工具多數針對開發者或高資產淨值客戶，普通用戶缺乏簡單有效的解決方案。

# Target Users & Personas
- **主要用戶**：普通加密貨幣持有者，年齡在25至55歲之間，對技術不熟悉，擁有一定的數位資產。
- **次要用戶**：加密貨幣的受益人，可能是持有者的家人或朋友，對加密貨幣一無所知。

# User Pain Points
1. 不知道如何安全地將加密資產傳遞給家人。
2. 缺乏簡單的指導，無法理解如何訪問和管理加密資產。
3. 擔心安全性，對存儲訪問指令的平臺缺乏信任。

# Value Proposition
CryptoLegacy 提供一個簡單且安全的平臺，讓加密貨幣持有者能夠快速設置資產繼承計劃，並生成易於理解的指導說明，幫助非技術性受益人順利獲取資產。

# Core Use Cases & User Stories
1. **用戶故事**：作為一名加密貨幣持有者，我希望能夠在幾分鐘內設置我的資產繼承計劃，這樣我的家人就能在我不在時獲得我的資產。
2. **用戶故事**：作為一名受益人，我希望能夠按照簡單的步驟獲取加密資產，這樣我就不需要依賴專業人士的幫助。

# MVP Scope
- **Must-have**:
  - 單一錢包整合。
  - 繼承計劃設置功能。
  - 生成簡單的步驟指導說明。
  - 死亡開關功能，當持有者不活躍時觸發指令發送。

- **Nice-to-have**:
  - 多錢包整合。
  - 自定義指導說明編輯功能。
  - 社交分享功能，讓用戶可以分享他們的計劃。

# User Flow Overview
1. 用戶註冊並登入 CryptoLegacy。
2. 用戶選擇錢包整合並設置繼承計劃。
3. 系統生成步驟指導說明並發送給指定受益人。
4. 用戶完成設置後，系統啟用死亡開關功能。

# Basic System Architecture
- **前端**：React.js
- **後端**：Node.js + Express
- **數據庫**：MongoDB
- **安全性**：使用 AES 加密技術存儲用戶數據。

# Monetization Hypothesis
CryptoLegacy 將以月費模式收費，基本計劃為每月10美元，涵蓋單一錢包，進階計劃為每月30美元，涵蓋多個錢包。潛在的合作夥伴包括加密稅務會計師和遺產規劃律師。

# Success Metrics
1. 用戶註冊數量。
2. 每月活躍用戶數。
3. 成功完成資產繼承的用戶比例。
4. 用戶滿意度調查結果。

# Risks & Assumptions
- **風險**：用戶對於存儲敏感數據的平臺缺乏信任。
- **假設**：普通加密貨幣持有者願意為簡單的繼承解決方案支付月費。
