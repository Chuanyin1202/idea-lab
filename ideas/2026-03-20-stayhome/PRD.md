# 產品名稱
StayHome

# One-line Pitch
提供居家安全評估，幫助長者安全地在家中生活，預防意外跌倒。

# Background & Problem Statement
隨著全球老年人口的增加，越來越多的長者希望在家中安享晚年。然而，根據統計，每四位長者中就有一位在一年內會發生跌倒，且跌倒所造成的住院費用平均高達35,000美元。許多潛在的危險因素如鬆動的地毯、缺乏扶手的浴室等，往往在家人眼中是不可見的，直到意外發生後才會被注意到。因此，提供一個專業的居家安全評估服務，能夠有效降低長者在家中生活的風險。

# Target Users & Personas
1. **長者**：希望在家中安全生活的老年人。
2. **家庭成員**：關心長者安全的子女或親屬。
3. **醫療專業人士**：如出院計劃員、老年法律顧問等，尋求能夠提供給家庭的具體建議。

# User Pain Points
- 長者在家中面臨的潛在危險無法被及時發現。
- 家庭成員缺乏專業知識，無法有效評估居家安全。
- 對於必要的修繕和改進缺乏明確的成本預算和承包商資訊。

# Value Proposition
StayHome 提供專業的居家安全評估服務，透過專業評估師的檢查，識別潛在危險並提供具體的修繕建議及成本預算，協助家庭成員有效改善長者居住環境的安全性。

# Core Use Cases & User Stories
1. **用戶故事 1**：作為一位長者，我希望能夠獲得專業的居家安全評估，以便了解我家中是否存在潛在的跌倒風險。
2. **用戶故事 2**：作為長者的子女，我希望能夠簡單地安排一個居家安全評估，並獲得具體的改善建議和承包商資訊。
3. **用戶故事 3**：作為醫療專業人士，我希望能夠向家庭推薦一個可靠的居家安全評估服務，以幫助他們改善長者的居住安全。

# MVP Scope (Must-have vs Nice-to-have)
## Must-have
- 專業的居家安全評估服務。
- 評估報告生成，包括風險評分、修繕建議及成本預算。
- 與承包商的聯繫功能。

## Nice-to-have
- 定期的安全監控服務，提供季度檢查。
- 用戶評價系統，幫助選擇承包商。

# User Flow Overview
1. 用戶訪問 StayHome 網站或應用程式。
2. 用戶填寫評估申請表。
3. 評估師安排上門評估。
4. 評估師進行檢查並拍攝照片。
5. 生成評估報告並發送給用戶及家庭成員。
6. 提供承包商聯繫資訊，協助用戶進行修繕。

# Basic System Architecture
- **前端**：React 或 Vue.js
- **後端**：Node.js + Express
- **數據庫**：MongoDB
- **影像處理**：使用 AWS S3 儲存照片，並利用 AI 技術生成報告。
- **支付系統**：Stripe 或 PayPal 用於處理支付。

# Monetization Hypothesis
- 每次評估收費 $199 至 $299。
- 當承包商完成修繕工作時，收取 $500 至 $1,000 的推薦費用。
- 提供每月 $25 的持續監控服務。

# Success Metrics
- 每月完成的評估數量。
- 家庭聘請承包商的比例。
- 客戶滿意度評價。
- 每月訂閱服務的用戶數量。

# Risks & Assumptions
- 風險：市場競爭可能影響服務的接受度。
- 假設：家庭成員願意為長者的安全投資，並且承包商能夠提供高質量的服務。
