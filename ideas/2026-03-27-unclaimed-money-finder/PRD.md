# 產品名稱
Unclaimed Money Finder

# One-line Pitch
一個幫助人們輕鬆尋找和索回未索取資金的平台。

# Background & Problem Statement
在許多國家，未索取的資金（如舊銀行賬戶、稅款退款、保險賠償等）存在於政府和金融機構的數據庫中，但由於缺乏有效的搜索工具，普通人很難找到這些資金。不同的系統使用不同的搜索邏輯和名稱匹配規則，導致人們在查詢後往往放棄。這使得大量的資金無法被索回，造成了資源的浪費。

# Target Users & Personas
- **個人用戶**：希望尋找未索取資金的普通民眾，特別是那些在過去有過銀行賬戶或保險的用戶。
- **財務顧問**：需要幫助客戶尋找未索取資金的專業人士。
- **律師**：專注於遺產管理和資產索回的法律專業人士。

# User Pain Points
- 難以理解不同政府機構的索回流程。
- 多個數據庫中信息不一致，搜索結果不可靠。
- 索回過程繁瑣，缺乏透明度和信任感。

# Value Proposition
Reclaimd 提供一個統一的平台，通過簡單的名字匹配和身份驗證流程，幫助用戶快速找到和索回未索取的資金。用戶只需輸入基本信息，系統將自動搜索並處理索回申請，節省時間和精力。

# Core Use Cases & User Stories
1. **用戶註冊**：用戶創建賬戶並提供基本信息。
2. **資金搜索**：用戶輸入姓名和出生日期，系統自動搜索多個數據庫。
3. **索回申請**：用戶選擇要索回的資金，系統引導用戶完成申請流程。
4. **狀態追蹤**：用戶可以隨時查看申請狀態和進度。

# MVP Scope (Must-have vs Nice-to-have)
## Must-have
- 名字匹配層，支持模糊邏輯和出生日期驗證。
- 手動提交前20個索回申請以學習流程。
- 基本的身份驗證功能。

## Nice-to-have
- 自動化索回申請流程。
- 多國語言支持。
- 用戶儀表板顯示索回進度和歷史記錄。

# User Flow Overview
1. 用戶註冊並登錄。
2. 輸入姓名和出生日期進行搜索。
3. 查看搜索結果，選擇要索回的資金。
4. 提交索回申請，系統引導用戶完成身份驗證。
5. 用戶在儀表板上追蹤申請進度。

# Basic System Architecture
- **前端**：React.js
- **後端**：Node.js + Express
- **數據庫**：MongoDB
- **身份驗證**：OAuth 2.0
- **數據抓取**：使用 Python 的 Scrapy 庫從政府數據庫抓取數據。

# Monetization Hypothesis
- 對於成功索回的資金收取12%的手續費。
- 提供每月$9.99的資金監控服務，讓用戶持續追蹤未索取資金。

# Success Metrics
- 每月活躍用戶數（MAU）。
- 成功索回的資金總額。
- 用戶滿意度和NPS（淨推廣值）。

# Risks & Assumptions
- **風險**：法律和合規問題，可能影響索回流程的自動化。
- **假設**：用戶對於使用自動化工具尋找未索取資金持開放態度，並願意支付服務費用。
