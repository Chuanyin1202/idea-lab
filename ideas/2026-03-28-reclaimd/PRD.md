# 產品名稱
Reclaimd

# One-line Pitch
幫助用戶輕鬆尋找和索回未認領的資金，從而獲得他們應得的資源。

# Background & Problem Statement
許多人在政府和金融機構的未認領資金數據庫中擁有資金，但由於搜尋過程繁瑣且不易，很多人放棄了尋找。不同的系統有不同的搜尋邏輯和名稱匹配規則，這使得用戶難以找到自己的資金。Reclaimd 將簡化這一過程，通過統一的搜尋平台幫助用戶找到並索回未認領的資金。

# Target Users & Personas
- **個人用戶**：希望尋找未認領資金的普通民眾，對技術不太熟悉。
- **財務顧問**：為客戶提供財務建議的專業人士，需幫助客戶索回資金。
- **遺產律師**：處理遺產事務，需為客戶尋找未認領資金。

# User Pain Points
- 繁瑣的搜尋過程導致用戶放棄。
- 不同的政府機構有不同的索賠流程，缺乏統一的資訊。
- 用戶對身份驗證過程感到不安，擔心資料安全。

# Value Proposition
Reclaimd 提供一個簡單易用的平台，通過一次搜尋即可查找多個國家的未認領資金，並自動處理索賠過程，讓用戶無需再為繁瑣的流程而煩惱。

# Core Use Cases & User Stories
1. **用戶搜尋未認領資金**：
   - 作為一名用戶，我希望能夠通過輸入我的姓名和出生日期來搜尋未認領的資金，以便找到我可能擁有的資金。
   
2. **自動索賠處理**：
   - 作為一名用戶，我希望在找到未認領資金後，能夠自動提交索賠申請，這樣我就不需要手動處理繁瑣的文件。

3. **狀態追蹤**：
   - 作為一名用戶，我希望能夠隨時查看我的索賠狀態，以便了解進度和預期的處理時間。

# MVP Scope
- **Must-have**:
  - 統一的名稱匹配層，支持模糊邏輯和出生日期驗證。
  - 手動提交前 20 個索賠以了解流程。
  - 基本的身份驗證系統。
  
- **Nice-to-have**:
  - 自動化索賠提交。
  - 多國資料庫的擴展搜尋功能。
  - 用戶界面的進一步優化和美化。

# User Flow Overview
1. 用戶註冊並登入平台。
2. 輸入姓名和出生日期進行搜尋。
3. 系統顯示找到的未認領資金。
4. 用戶選擇索賠並提交相關資料。
5. 系統進行身份驗證並提交索賠。
6. 用戶可隨時查詢索賠進度。

# Basic System Architecture
- **前端**：React.js
- **後端**：Node.js + Express
- **數據庫**：MongoDB
- **身份驗證**：OAuth 2.0
- **雲端服務**：AWS（用於數據存儲和處理）

# Monetization Hypothesis
- 收取 12% 的回收資金手續費。
- 提供每月 $9.99 的資金監控服務，無回收則不收費。

# Success Metrics
- 每月活躍用戶數 (MAU)。
- 成功索回的資金總額。
- 用戶滿意度調查結果。

# Risks & Assumptions
- **風險**：用戶對身份驗證的信任度不足，可能會影響使用率。
- **假設**：用戶對未認領資金的需求存在潛在市場，且願意為此支付手續費。
