# 產品名稱
Restamp

## One-line Pitch
一個針對持牌專業人士的持續教育市場，提供經過驗證的課程評價和合規追蹤工具。

## Background & Problem Statement
許多持牌專業人士在更新執照時面臨著繁瑣的持續教育要求。不同州對於課程的要求各不相同，專業人士常常在選擇課程時只考量價格與時長，而忽略了課程的質量。這導致他們可能選擇了不合格的課程，最終影響執照的續期。此外，現有的課程評價系統缺乏可信度，專業人士對於課程的選擇感到困惑。

## Target Users & Personas
1. **護士**：需要每兩年更新執照，對於課程的質量和合規性有高要求。
2. **房地產經紀人**：根據州的不同，每2到4年需要更新執照，對於課程的便捷性和價格敏感。
3. **法院指定監護人**：需要定期提交報告並確保合規，對於工具的需求強烈。

## User Pain Points
- 難以找到符合州要求的合格課程。
- 課程評價不可信，無法確保選擇的課程質量。
- 更新執照的過程繁瑣，容易錯過截止日期。
- 缺乏有效的工具來追蹤已完成的學分和合規性。

## Value Proposition
Restamp 提供一個集成平台，專業人士可以在此找到經過驗證的課程評價，並且能夠追蹤自己的學分和合規性。平台不僅幫助用戶選擇合適的課程，還能在執照更新前發送提醒，減少用戶的焦慮和時間成本。

## Core Use Cases & User Stories
1. **用戶註冊**：作為一名護士，我希望能夠註冊並填寫我的執照信息，以便獲得個性化的課程推薦。
2. **課程搜尋**：作為一名房地產經紀人，我希望能夠根據我的州和需求搜尋合格的課程。
3. **課程評價**：作為一名專業人士，我希望能夠查看其他持牌專業人士對課程的評價，以便做出明智的選擇。
4. **合規追蹤**：作為一名法院指定監護人，我希望能夠追蹤我的學分並在執照到期前收到提醒。

## MVP Scope
### Must-have
- 用戶註冊和登錄功能
- 課程搜尋和過濾功能
- 課程評價系統（經過驗證的用戶評價）
- 學分追蹤和合規提醒功能

### Nice-to-have
- 課程提供者的後台管理系統
- 社群討論區，讓用戶分享經驗
- 數據分析儀表板，顯示用戶的學習進度

## User Flow Overview
1. 用戶註冊並填寫執照信息。
2. 用戶根據需求搜尋課程。
3. 用戶查看課程評價並選擇報名。
4. 平台追蹤學分並在到期前發送提醒。

## Basic System Architecture
- **前端**：React.js
- **後端**：Node.js + Express
- **數據庫**：MongoDB
- **身份驗證**：JWT
- **雲服務**：AWS S3（文件存儲）

## Monetization Hypothesis
- 專業人士每月支付 $29 的訂閱費用以獲得課程評價和合規追蹤服務。
- 課程提供者支付每月 $200 至 $500 的廣告費用，並對每個註冊收取 15% 的手續費。

## Success Metrics
- 每月活躍用戶數（MAU）
- 用戶留存率
- 課程評價數量
- 訂閱收入增長率

## Risks & Assumptions
- 風險：用戶對於課程評價的信任度不高，可能影響平台的使用率。
- 假設：專業人士願意為合規追蹤和課程評價支付訂閱費用。
