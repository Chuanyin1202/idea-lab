# 產品名稱
Classmold

# One-line Pitch
一個讓教師能夠根據課程計劃創建互動活動的工具，提升學生的學習體驗。

# Background & Problem Statement
在K-12教育中，教師通常需要設計課程並選擇教學材料，但當課程需要互動活動時，教師面臨的選擇有限。現有的互動活動大多集中在科學領域，缺乏針對歷史、經濟學和社會研究等學科的專門工具。這使得教師無法有效地將互動學習融入他們的課程中，從而影響學生的參與度和學習效果。

# Target Users & Personas
- **教師**：K-12學校的教師，尤其是教授歷史、經濟學和社會研究的教師。
- **學校管理者**：負責課程設計和教學質量的學校管理者，尋求提升學校教育質量的工具。
- **教育技術專家**：對教育技術工具有興趣，並希望將其整合到現有課程中的專業人士。

# User Pain Points
- 缺乏針對特定學科的互動教學資源。
- 現有工具的範圍有限，無法滿足教師的需求。
- 教師在設計互動活動時缺乏支持和指導。
- 學生參與度低，影響學習效果。

# Value Proposition
Classmold提供一個平台，讓教師能夠根據自己的課程計劃創建專屬的互動活動。通過AI建議和學科特定的模板，教師可以輕鬆設計出符合課程需求的互動學習體驗，從而提升學生的參與度和學習效果。

# Core Use Cases & User Stories
1. **教師設計互動活動**：
   - 作為一名歷史教師，我希望能夠根據課程計劃創建互動時間軸，讓學生更好地理解歷史事件的因果關係。
   
2. **AI建議活動結構**：
   - 作為一名經濟學教師，我希望AI能根據我上傳的課程內容建議合適的互動活動結構，以便我能夠有效地傳達供需模型。

3. **活動分享與重混**：
   - 作為一名教師，我希望能夠將我創建的互動活動分享給其他教師，並從他們那裡獲得靈感和改進建議。

# MVP Scope (Must-have vs Nice-to-have)
## Must-have
- 課程計劃上傳功能。
- 學科特定的互動活動模板。
- AI建議活動結構的功能。
- 活動分享與重混功能。
- 基本的分析工具，幫助教師評估學生參與度。

## Nice-to-have
- 高級分析報告功能，提供更深入的學習數據。
- 白標版本供學校使用。
- 整合學習管理系統（LMS）的功能。

# User Flow Overview
1. 教師註冊並登錄Classmold平台。
2. 教師上傳課程計劃。
3. 系統根據上傳的內容提供學科特定的活動模板和AI建議。
4. 教師選擇模板並自定義互動活動。
5. 教師發布活動並分享至平台。
6. 學生參與互動活動並提供反饋。
7. 教師查看分析報告，評估活動效果。

# Basic System Architecture
- **前端**：React.js
- **後端**：Node.js + Express
- **數據庫**：MongoDB
- **AI建議引擎**：Python + TensorFlow
- **雲服務**：AWS或Google Cloud（用於存儲和計算）

# Monetization Hypothesis
- 每位教師每月收費25美元，或每年3000美元的學校區域授權費用。
- 通過教師分享的活動建立市場，並從中獲取佣金。
- 提供白標版本的高級服務，收取額外費用。

# Success Metrics
- 每月活躍用戶數（MAU）。
- 教師創建的互動活動數量。
- 學生參與互動活動的比例。
- 客戶滿意度調查結果。

# Risks & Assumptions
- 假設教師對互動學習工具的需求持續增長。
- 假設AI建議的活動結構能夠有效提升學生參與度。
- 風險包括市場競爭和技術實現的挑戰。
