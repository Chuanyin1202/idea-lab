# 產品名稱
Classmold

# One-line Pitch
為K-12教師提供互動式課程活動建構工具，讓教學更具吸引力和互動性。

# Background & Problem Statement
目前的教育工具多數集中於科學領域，缺乏針對歷史、經濟學和社會研究等科目的互動式學習資源。教師在設計課程時，常常無法找到合適的互動活動來增強學生的學習體驗，這導致學生的參與度和學習效果下降。Classmold旨在解決這一問題，幫助教師根據課程需求自訂互動活動。

# Target Users & Personas
- **教師**：K-12學校的各科教師，尋求創新的教學方法以提高學生的參與度。
- **學校管理者**：關心教學質量和學生表現的校方決策者。
- **教育科技專家**：對教育工具的效果和數據分析有興趣的專業人士。

# User Pain Points
1. 現有互動學習工具多數集中於科學，缺乏其他科目的支持。
2. 教師難以找到合適的互動活動來符合課程需求。
3. 教學資源的缺乏導致學生參與度低，學習效果不佳。
4. 教師需要時間來設計和調整互動活動，影響教學效率。

# Value Proposition
Classmold提供一個平台，讓教師能夠根據自己的課程計劃創建互動活動，並利用AI建議來優化活動設計。透過共享和重混的市場，教師可以輕鬆獲得和分享高質量的教學資源，進而提高學生的學習參與度和成效。

# Core Use Cases & User Stories
1. **用戶故事1**：作為一名歷史教師，我希望能夠根據課程內容創建互動式時間線，以便學生能更好地理解歷史事件的因果關係。
2. **用戶故事2**：作為一名經濟學教師，我希望能夠使用模板來建立供需模型，讓學生能夠直觀地學習經濟原則。
3. **用戶故事3**：作為學校管理者，我希望能夠查看教師使用Classmold後的教學效果數據，以便做出更好的資源分配決策。

# MVP Scope (Must-have vs Nice-to-have)
## Must-have
- 互動活動建構工具
- 主題模板庫（歷史、經濟學、社會研究）
- AI建議功能
- 教師共享市場
- 基本數據分析功能

## Nice-to-have
- 白標版本供學校使用
- 更高級的數據分析和報告功能
- 多語言支持
- 社交功能（教師之間的互動）

# User Flow Overview
1. 教師註冊並登錄Classmold。
2. 上傳課程計劃並選擇主題模板。
3. 使用互動活動建構工具設計活動。
4. AI提供建議並協助優化活動。
5. 教師分享活動至市場，其他教師可以使用或重混。
6. 系統收集數據並提供分析報告給教師和學校管理者。

# Basic System Architecture
- **前端**：React.js
- **後端**：Node.js + Express
- **數據庫**：MongoDB
- **AI建議系統**：Python + TensorFlow
- **雲端服務**：AWS或Google Cloud
- **分析工具**：Google Analytics或自建數據分析模組

# Monetization Hypothesis
- 每位教師每月收費25美元，或每所學校每年收費3000美元的區域授權。
- 透過教師共享市場獲取額外收入。
- 提供白標版本給學校，收取額外費用。

# Success Metrics
- 每月活躍用戶數（MAU）
- 教師創建的互動活動數量
- 學生參與度提升的百分比
- 學校的訂閱轉換率

# Risks & Assumptions
- 風險：教師對新工具的接受度低，可能影響使用率。
- 假設：教師願意為提高教學質量支付額外費用。
- 風險：市場上競爭對手的出現，可能影響Classmold的市場份額。
