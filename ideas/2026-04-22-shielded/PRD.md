# 產品名稱
Shielded

# One-line Pitch
一個利用 AI 技術幫助小型企業保險索賠的工具，確保業主獲得應有的賠償。

# Background & Problem Statement
許多小型企業在遭遇火災、屋頂倒塌或水災等災難時，會依賴商業保險來減輕損失。然而，保險公司往往在索賠過程中提供低於預期的賠償金，導致企業主不得不承擔短缺的費用。雖然企業主可以聘請公眾調整員來幫助處理索賠，但這會產生高達15%的費用，對於資金緊張的小型企業來說，這是一筆不小的負擔。因此，市場需要一個能夠幫助企業主理解保險條款並有效處理索賠的工具。

# Target Users & Personas
- **小型企業主**：如餐廳、零售店和服務業，經常面臨自然災害或意外事件。
- **保險代理人**：需要提供更好的客戶服務，幫助客戶理解保險條款。
- **公眾調整員**：需要工具來提高工作效率，減少手動文檔處理。

# User Pain Points
1. 不清楚保險政策的具體條款和限制。
2. 索賠過程繁瑣且耗時，缺乏有效的指導。
3. 難以確定應索賠的金額及所需的文件。
4. 高額的公眾調整員費用使得小型企業無法承擔。

# Value Proposition
Shielded 提供一個簡單易用的 AI 工具，幫助小型企業主上傳保險文件，快速識別保險範圍和索賠條件，並在發生索賠時提供所需的文檔支持，從而提高索賠的成功率，減少企業主的經濟損失。

# Core Use Cases & User Stories
1. **上傳保險文件**：作為一名小型企業主，我希望能夠輕鬆上傳我的保險文件，讓 AI 幫我分析保險條款。
2. **索賠指導**：作為一名企業主，在發生災難後，我希望能獲得具體的索賠指導，讓我知道應該如何收集和提交文件。
3. **追蹤索賠進度**：作為一名企業主，我希望能夠隨時查看我的索賠進度，了解何時能獲得賠償。

# MVP Scope (Must-have vs Nice-to-have)
## Must-have
- 文件上傳功能
- AI 條款分析
- 索賠指導流程
- 用戶賬戶管理

## Nice-to-have
- 索賠進度追蹤
- 整合保險公司 API
- 提供公眾調整員推薦

# User Flow Overview
1. 用戶註冊並登入系統。
2. 上傳商業保險文件。
3. AI 分析文件並生成報告。
4. 提供索賠指導，指示用戶收集所需文件。
5. 用戶提交索賠請求。

# Basic System Architecture
- **前端**：React.js
- **後端**：Node.js + Express
- **資料庫**：MongoDB
- **AI 模型**：Python + TensorFlow
- **雲服務**：AWS S3（文件儲存）、AWS Lambda（無伺服器運算）

# Monetization Hypothesis
- 訂閱制收費：提供不同層級的服務計劃，根據用戶需求收取月費或年費。
- 按次計費：針對每次索賠請求收取手續費。

# Success Metrics
- 每月活躍用戶數（MAU）
- 用戶留存率
- 每月索賠請求數量
- 用戶滿意度調查結果

# Risks & Assumptions
- **風險**：用戶對 AI 分析結果的信任度不足，可能影響使用率。
- **假設**：小型企業主願意為提高索賠成功率支付訂閱費用。
