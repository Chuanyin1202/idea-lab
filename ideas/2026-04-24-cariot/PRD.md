# 產品名稱
Cariot

# One-line Pitch
一款專為老年人設計的交通服務應用，確保家庭成員隨時掌握長者的出行狀況。

# Background & Problem Statement
隨著人口老齡化，越來越多的老年人需要依賴交通服務來維持他們的獨立性，進行醫療約診、藥品取貨和日常購物。然而，雖然有許多應用程式提供預訂服務，但確認司機是否準時到達、約診是否如期進行以及回程是否安排妥當，卻成為了遠在他鄉的家庭成員的難題。這種情況導致了家庭成員的焦慮和不安。

# Target Users & Personas
- **老年人**：需要定期交通服務的使用者，對科技的接受度不一。
- **家庭成員**：如子女或親屬，負責關注老年人的出行安全，通常居住在不同城市或地區。

# User Pain Points
- 難以確認老年人是否安全抵達目的地。
- 多個應用程式和服務的使用造成混亂。
- 老年人對科技的操作困難，無法輕鬆預訂交通服務。
- 家庭成員需要花費大量時間進行溝通和確認。

# Value Proposition
Cariot 提供一個簡單易用的應用程式，讓老年人能夠輕鬆預訂交通服務，同時讓家庭成員能即時獲取出行狀況更新，減少焦慮，提升出行安全感。

# Core Use Cases & User Stories
1. **預訂交通服務**：
   - 作為一位老年人，我希望能夠透過語音命令或簡單的按鈕預訂交通服務，以便我能輕鬆出行。
   
2. **實時更新**：
   - 作為一位家庭成員，我希望能夠即時收到老年人出行的狀態更新，以便我能夠隨時掌握他們的安全情況。

3. **日曆同步**：
   - 作為一位老年人，我希望我的醫療約診能夠自動同步到應用程式中，讓我不必擔心忘記預約。

# MVP Scope (Must-have vs Nice-to-have)
## Must-have
- 基本的交通預訂功能。
- 實時通知功能，讓家庭成員獲得更新。
- 簡單的用戶界面，適合老年人使用。
- 醫療日曆同步功能。

## Nice-to-have
- 語音助手功能，提供更便捷的預訂方式。
- 社交功能，讓家庭成員之間可以互相交流。
- 數據分析，提供出行模式的報告。

# User Flow Overview
1. 老年人打開應用程式。
2. 選擇預訂交通服務。
3. 確認目的地和時間。
4. 系統發送確認通知給家庭成員。
5. 司機到達後，家庭成員收到實時更新。
6. 完成行程後，家庭成員收到行程結束通知。

# Basic System Architecture
- **前端**：React Native（移動應用）
- **後端**：Node.js + Express.js
- **數據庫**：MongoDB
- **即時通知**：Firebase Cloud Messaging
- **API整合**：第三方交通服務API

# Monetization Hypothesis
- 透過收取每次預訂的手續費來獲利。
- 提供高級功能的訂閱服務，例如更詳細的出行報告和更快的客戶支持。

# Success Metrics
- 每月活躍用戶數（MAU）。
- 每次預訂的平均收入（ARPU）。
- 用戶滿意度調查結果。
- 服務使用頻率。

# Risks & Assumptions
- **風險**：老年人對新技術的接受度低，可能影響使用率。
- **假設**：家庭成員願意為提升老年人出行安全支付額外費用。
