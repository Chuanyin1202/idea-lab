# 產品名稱
Cariot

# One-line Pitch
一個專為老年人設計的交通服務應用，確保家庭能隨時掌握長者的出行狀況。

# Background & Problem Statement
隨著人口老齡化，老年人對於交通服務的需求日益增加。他們需要定期的醫療約診、藥品取貨和日常購物，但在這些出行中，家庭成員常常無法及時獲知他們的安全狀況。現有的交通應用雖然能夠輕鬆預約，但卻無法提供實時的確認和跟蹤，這造成了家庭成員的焦慮和不便。

# Target Users & Personas
- **老年人**：需要獨立出行的長者，對科技的接受度不一。
- **家庭成員**：關心長者出行安全的子女或親屬，通常居住在不同的城市。
- **醫療機構**：需要安排患者交通的醫療服務提供者。

# User Pain Points
1. 無法實時確認長者是否安全到達目的地。
2. 預約過程繁瑣，對於不熟悉科技的老年人來說尤其困難。
3. 家庭成員需要不斷透過電話或訊息確認長者的狀況，增加了溝通成本。

# Value Proposition
Cariot 提供一個簡單易用的交通服務平台，專為老年人設計，並且能夠實時更新行程狀況，讓家庭成員隨時掌握長者的出行情況，減少焦慮與不便。

# Core Use Cases & User Stories
1. **預約交通**：老年人可以透過語音指令或大型按鈕輕鬆預約交通服務。
   - *用戶故事*: 作為一名老年人，我希望能夠用語音預約我的醫療約診交通，讓我不必擔心操作複雜的應用。
   
2. **行程跟蹤**：家庭成員能夠在應用中實時查看長者的行程狀況。
   - *用戶故事*: 作為一名子女，我希望能夠在應用中看到我母親的行程狀況，這樣我就能放心她的安全。

3. **自動通知**：當長者到達目的地後，自動發送通知給家庭成員。
   - *用戶故事*: 作為一名家庭成員，我希望在我父親到達醫院後能即時收到通知，這樣我就能安心。

# MVP Scope (Must-have vs Nice-to-have)
## Must-have
- 語音預約功能
- 大型按鈕界面
- 行程實時跟蹤
- 自動通知系統

## Nice-to-have
- 醫療日曆同步功能
- 家庭成員群組聊天功能
- 評價系統以提高服務質量

# User Flow Overview
1. 老年人打開應用，使用語音或按鈕預約交通。
2. 系統確認預約並發送通知給家庭成員。
3. 司機接單並接送老年人。
4. 到達目的地後，自動發送通知給家庭成員。

# Basic System Architecture
- **前端**：React Native (移動應用)
- **後端**：Node.js + Express (API)
- **數據庫**：MongoDB (用戶和行程數據)
- **實時通知**：Firebase Cloud Messaging
- **語音識別**：Google Cloud Speech-to-Text

# Monetization Hypothesis
- 訂閱制：家庭成員可選擇月費或年費訂閱服務，獲取額外功能如行程歷史記錄。
- 交易佣金：每次成功預約收取一定比例的佣金。

# Success Metrics
- 每月活躍用戶數 (MAU)
- 用戶留存率
- 每次行程的平均滿意度評分
- 家庭成員的通知響應率

# Risks & Assumptions
- **風險**：長者對科技的接受度可能影響使用率。
- **假設**：家庭成員對於長者出行的關心將驅動他們使用此服務。
