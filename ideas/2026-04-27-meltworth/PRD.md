# 產品名稱
Meltworth

# One-line Pitch
即時廢金屬價格追蹤，為承包商提供透明的報價，幫助他們最大化收益。

# Background & Problem Statement
在廢金屬行業中，價格的透明度極低。廢料場每天都會隨意設定價格，並且不會公開這些價格，導致承包商在運送金屬時無法獲得公平報價。這使得承包商在每次交易中損失20-30%的潛在收益。Meltworth的目標是透過收集和顯示廢料場的即時價格，來解決這一問題。

# Target Users & Personas
目標用戶主要為承包商，包括：
- **建築承包商**：經常需要處理拆除和重建項目，並運送金屬廢料。
- **小型企業主**：經營金屬回收業務，需要快速獲取市場價格以提高利潤。
- **個體戶**：偶爾運送廢金屬，對價格敏感。

# User Pain Points
1. 缺乏透明的價格資訊，導致承包商無法比較不同廢料場的報價。
2. 每次運送後需要花費大量時間進行電話詢價。
3. 無法即時獲取市場價格，影響決策效率。
4. 對於金屬種類的識別和估價存在困難。

# Value Proposition
Meltworth提供一個即時的廢金屬價格追蹤平台，讓承包商能夠快速獲取周圍廢料場的報價，並通過拍攝貨物照片來自動識別金屬類型，從而提高交易效率和收益。

# Core Use Cases & User Stories
1. **用戶故事1**：作為一名建築承包商，我希望能夠在完成拆除後，立即查看附近廢料場的報價，以便快速決策。
2. **用戶故事2**：作為一名小型企業主，我希望能夠透過拍攝金屬廢料的照片來獲取報價，這樣我就不需要逐一聯繫廢料場。
3. **用戶故事3**：作為一名個體戶，我希望能夠隨時隨地使用手機查看廢金屬的市場價格，這樣我可以在最佳時機出售我的廢料。

# MVP Scope (Must-have vs Nice-to-have)
## Must-have
- 實時廢料場價格數據收集及顯示功能
- 拍照識別金屬類型的功能
- 用戶註冊及登錄系統
- 基本的價格比較功能

## Nice-to-have
- 歷史價格趨勢分析
- 用戶評價及反饋系統
- 廢料場位置導航功能
- 自動通知用戶價格變動

# User Flow Overview
1. 用戶下載並安裝Meltworth應用程式。
2. 用戶註冊並登錄。
3. 用戶拍攝金屬廢料的照片。
4. 應用程式自動識別金屬類型並查詢附近廢料場的即時價格。
5. 用戶查看報價並選擇最佳選項進行交易。

# Basic System Architecture
- **前端**：React Native（移動應用）
- **後端**：Node.js + Express
- **數據庫**：MongoDB
- **第三方API**：廢料場價格數據API
- **圖像識別**：TensorFlow或Google Vision API

# Monetization Hypothesis
Meltworth可以透過以下方式獲利：
1. 向廢料場收取廣告費用，讓他們在平台上展示價格。
2. 提供高級會員服務，讓用戶獲得更多功能，如歷史價格分析和專屬報價。
3. 透過交易手續費，當用戶通過平台完成交易時收取一定比例的手續費。

# Success Metrics
- 每月活躍用戶數（MAU）
- 用戶留存率
- 平均每次交易的收益提升百分比
- 廢料場的報價更新頻率

# Risks & Assumptions
- **風險**：廢料場可能不願意提供即時價格數據，影響平台的準確性。
- **假設**：承包商對於使用移動應用程式獲取報價的接受度較高，並願意改變傳統的詢價方式。
