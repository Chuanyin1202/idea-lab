# 產品名稱
Magpie

# One-line Pitch
一個自動標記的工具，為設計師提供無縫的靈感捕捉與品牌智能管理。

# Background & Problem Statement
在設計行業中，設計師經常面臨靈感來源的分散與管理困難。現有的工具如Pinterest、Are.na等，主要針對個人用戶，無法滿足設計團隊在多客戶環境中的需求。設計師需要一個能夠自動標記和組織靈感的解決方案，以便於在不同客戶之間快速切換和查找資料。

# Target Users & Personas
- **設計師**：在設計工作中需要快速捕捉和組織靈感的專業人士。
- **設計經理**：負責管理設計團隊，需追蹤團隊的工作進度和品牌資產。
- **自由職業者**：在多個客戶之間工作，需要高效管理不同品牌的靈感和參考資料。

# User Pain Points
1. 靈感來源分散在不同的應用和文件夾中，查找效率低下。
2. 無法快速標記和組織靈感，導致重複工作。
3. 客戶之間的品牌資產無法有效隔離，造成混淆。
4. 現有工具無法提供視覺相似性搜索，影響創作效率。

# Value Proposition
Magpie通過自動標記和同步功能，幫助設計師高效捕捉靈感，並將其整合到設計工具中，提升工作效率和品牌智能管理。用戶可以在不同客戶之間輕鬆切換，並快速找到所需的參考資料。

# Core Use Cases & User Stories
1. **靈感捕捉**：設計師在瀏覽網頁時，使用瀏覽器擴展一鍵捕捉靈感，並自動標記。
   - *使用者故事*：作為一名設計師，我希望能夠快速捕捉靈感，這樣我就不會錯過任何創意。
   
2. **品牌資產管理**：設計經理可以查看不同客戶的品牌資產，並確保資料的隔離性。
   - *使用者故事*：作為設計經理，我希望能夠管理各個客戶的品牌資料，這樣我可以確保資料不會混淆。

3. **視覺相似性搜索**：用戶可以根據視覺特徵快速查找相關靈感。
   - *使用者故事*：作為一名設計師，我希望能夠根據視覺特徵查找靈感，這樣我可以更快地找到合適的參考資料。

# MVP Scope (Must-have vs Nice-to-have)
## Must-have
- 自動標記功能
- 與Figma、Eagle、Are.na的同步功能
- 客戶特定的標籤命名空間
- 基本的視覺相似性搜索功能

## Nice-to-have
- 高級分析儀表板
- 團隊協作功能
- 自定義標籤和分類功能

# User Flow Overview
1. 用戶安裝Magpie瀏覽器擴展。
2. 用戶在瀏覽時捕捉靈感，Magpie自動標記並存儲。
3. 用戶選擇客戶，並將靈感同步到相應的設計工具。
4. 用戶可以使用視覺相似性搜索功能查找相關靈感。

# Basic System Architecture
- **前端**：React.js (瀏覽器擴展)
- **後端**：Node.js + Express
- **數據庫**：MongoDB (存儲用戶資料和靈感)
- **AI標記**：使用CLIP嵌入和Pinecone進行視覺相似性搜索
- **雲服務**：AWS或GCP進行數據存儲和處理

# Monetization Hypothesis
- 針對設計團隊的訂閱制收費，每位用戶每月$50-$100。
- 自由職業者的個人計劃每月$20，吸引底層用戶進行採用。

# Success Metrics
- 用戶留存率
- 每月活躍用戶數
- 客戶滿意度調查結果
- 平均查找時間的減少

# Risks & Assumptions
- 競爭對手可能會快速模仿我們的功能。
- 用戶對自動標記的準確性有高期望，若未能達到，可能會影響用戶採用率。
- 需確保數據安全性，以保護客戶的品牌資產和資料。
