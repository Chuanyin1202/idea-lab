# 產品名稱
Joistbook

# One-line Pitch
一個為建築檢查員提供即時建築法規參考的工具，幫助他們在現場快速做出正確決策。

# Background & Problem Statement
在建築檢查過程中，檢查員需要面對不同的建築法規，這些法規因地區而異，且經常會有地方性修訂。這導致檢查員在現場時，常常無法迅速回憶起正確的法規內容，可能會因為忘記某個修訂而錯過發現違規的機會。Joistbook旨在解決這一問題，提供一個即時的建築法規參考工具，幫助檢查員在現場快速獲取所需資訊。

# Target Users & Personas
- **建築檢查員**: 專業從事建築檢查的工作，需了解多個地區的建築法規。
- **建築承包商**: 需要確保其施工符合當地法規的建築專業人士。
- **城市規劃師**: 需要了解建築法規以進行有效的城市規劃。

# User Pain Points
1. 難以記憶不同地區的建築法規及其修訂。
2. 現場查詢法規時缺乏即時性，影響檢查效率。
3. 法規的複雜性使得檢查員容易出錯，導致潛在的法律責任。

# Value Proposition
Joistbook提供即時的建築法規查詢，結合自然語言處理技術，讓檢查員能夠快速獲取IRC引用、當地修訂及實用建議，提升檢查效率並降低錯誤率。

# Core Use Cases & User Stories
- **用例1**: 檢查員在現場遇到不確定的法規時，使用Joistbook查詢相關法規。
  - **用戶故事**: 作為一名建築檢查員，我希望能在現場快速查詢法規，以便在檢查過程中做出正確的決策。
  
- **用例2**: 檢查員需要了解當地的修訂及其影響。
  - **用戶故事**: 作為一名建築檢查員，我希望能夠獲得當地修訂的詳細資訊，這樣我可以更好地執行我的檢查工作。

# MVP Scope
## Must-have
- 自然語言查詢功能，能夠返回IRC引用、當地修訂及實用建議。
- 用戶界面設計，便於檢查員在現場使用。
- 數據庫包含主要的建築法規及其修訂。

## Nice-to-have
- 雲端同步功能，讓檢查員能夠在多個設備上訪問資料。
- 社群功能，讓用戶能夠分享經驗及建議。
- 數據分析功能，提供檢查趨勢報告。

# User Flow Overview
1. 用戶打開Joistbook應用程式。
2. 在搜索框中輸入相關問題或關鍵詞。
3. 系統返回相關的IRC引用、當地修訂及實用建議。
4. 用戶查看結果並根據建議進行檢查。

# Basic System Architecture
- **前端**: 使用React Native開發移動應用。
- **後端**: Node.js和Express處理API請求。
- **數據庫**: MongoDB儲存建築法規及修訂資料。
- **自然語言處理**: 使用Python和NLTK進行查詢解析。

# Success Metrics
- 每月活躍用戶數（MAU）。
- 用戶留存率。
- 用戶滿意度調查結果。
- 每月新增訂閱用戶數。

# Risks & Assumptions
- **風險**: 法規更新頻繁，需持續維護數據庫的準確性。
- **假設**: 檢查員願意使用移動應用程式進行法規查詢，並認為這能提高工作效率。
