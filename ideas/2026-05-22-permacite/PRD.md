# 產品名稱
Permacite

# One-line Pitch
一個創新的引用工具，幫助研究人員在資料來源消失之前，保存和格式化引用資料，確保學術工作的持久性。

# Background & Problem Statement
在學術研究中，引用資料的可靠性至關重要。然而，許多資料來源在短短幾年內就會失效，導致研究人員面臨資料鏈接失效的問題。這種現象稱為“鏈接腐爛”（link rot），使得原本的引用失去價值，影響學術作品的完整性。現有的引用工具僅負責格式化引用，並不解決資料來源消失的問題。

# Target Users & Personas
- **研究生**：需要大量引用資料來撰寫論文，對資料的持久性要求高。
- **學術研究人員**：經常進行長期研究，依賴於穩定的資料來源。
- **教師**：在教學中使用引用資料，需確保資料的可靠性。

# User Pain Points
- 資料來源經常失效，影響引用的有效性。
- 現有工具無法自動保存資料來源的快照。
- 研究過程中耗費大量時間在尋找和確認資料來源的有效性。

# Value Proposition
Permacite 提供一個簡單的解決方案，讓研究人員能夠在引用資料之前自動保存資料來源的快照，並在格式化引用時提供持久的鏈接，確保學術工作的完整性和可靠性。

# Core Use Cases & User Stories
1. **自動保存資料來源**：作為研究人員，我希望在引用資料時自動保存資料的快照，以便未來能夠隨時訪問。
2. **格式化引用**：作為研究人員，我希望能夠快速格式化引用，支持多種引用風格（如 APA、MLA），以便於我的學術寫作。
3. **檢索歷史快照**：作為研究人員，我希望能夠隨時檢索已保存的資料快照，以便確認引用的準確性。

# MVP Scope (Must-have vs Nice-to-have)
## Must-have
- 自動保存資料來源的快照功能
- 支持多種引用格式的引用生成器
- 用戶友好的界面和簡單的操作流程

## Nice-to-have
- 整合社交媒體和學術資料庫的引用功能
- 提供資料來源的可靠性評估工具
- 支持團隊協作功能，允許多位研究人員共享引用資料

# User Flow Overview
1. 用戶在 Permacite 中輸入或粘貼資料來源的鏈接。
2. 系統自動保存該鏈接的快照。
3. 用戶選擇引用格式並生成引用。
4. 用戶可以隨時檢索和查看已保存的快照。

# Basic System Architecture
- **前端**：React.js
- **後端**：Node.js + Express
- **數據庫**：MongoDB
- **快照存儲**：使用 Wayback Machine API 和自定義爬蟲
- **雲存儲**：AWS S3

# Monetization Hypothesis
Permacite 將採用訂閱制模式，提供免費試用期，之後收取月費或年費，並提供高級功能（如團隊協作和進階分析）作為額外收費項目。

# Success Metrics
- 註冊用戶數量
- 每月活躍用戶數
- 用戶留存率
- 收入增長率

# Risks & Assumptions
- **風險**：用戶對於資料隱私的擔憂可能影響使用意願。
- **假設**：研究人員對於引用資料的持久性有強烈需求，並願意為此支付服務費用。
