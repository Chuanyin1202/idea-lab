# 產品名稱
Hardhire

# One-line Pitch
為家庭提供承包商的安全評分，幫助用戶做出明智的雇用決策。

# Background & Problem Statement
在選擇承包商進行家庭裝修時，消費者往往依賴五顆星的評價來判斷承包商的服務質量。然而，這些評價並未考慮到承包商的安全記錄，包括過去的違規行為和事故。Hardhire旨在解決這一問題，通過提供承包商的安全評分，幫助用戶在雇用時考量安全因素。

# Target Users & Personas
1. **家庭業主**：希望進行裝修或維修的家庭，對承包商的安全記錄有高度關注。
2. **承包商**：希望提升自身的市場競爭力，並展示良好的安全記錄以吸引更多客戶。
3. **房地產經紀人**：需要為客戶提供可靠的承包商推薦，並考量安全因素。

# User Pain Points
- 難以獲取承包商的安全記錄和過去的違規行為。
- 現有的評價系統無法反映承包商的真實安全狀況。
- 對於家庭裝修的安全性缺乏透明度，增加了潛在風險。

# Value Proposition
Hardhire通過整合聯邦數據庫中的安全違規記錄，為每位承包商提供A到F的安全評分，讓家庭業主能夠快速了解承包商的安全記錄，從而做出更明智的雇用決策。

# Core Use Cases & User Stories
1. **家庭業主**：作為一名家庭業主，我希望能夠快速查詢承包商的安全評分，以便選擇最安全的選擇。
2. **承包商**：作為一名承包商，我希望能夠在平台上展示我的安全記錄，以吸引更多客戶。
3. **房地產經紀人**：作為一名房地產經紀人，我希望能夠向客戶提供安全可靠的承包商推薦。

# MVP Scope (Must-have vs Nice-to-have)
## Must-have
- 承包商的安全評分系統（A到F）。
- 整合聯邦數據庫的違規記錄查詢功能。
- 用戶界面，允許用戶查詢和比較承包商的安全評分。

## Nice-to-have
- 評價系統，讓用戶可以對承包商的服務進行評價。
- 承包商的詳細資料頁面，顯示其過去的項目和客戶評價。
- 通知功能，提醒用戶有關承包商的最新安全記錄。

# User Flow Overview
1. 用戶註冊並登錄Hardhire平台。
2. 用戶輸入承包商的名稱進行查詢。
3. 系統返回承包商的安全評分和違規記錄。
4. 用戶可以選擇保存或分享該信息。

# Basic System Architecture
- **前端**：React.js
- **後端**：Node.js + Express
- **數據庫**：MongoDB
- **API整合**：調用聯邦數據庫API以獲取承包商的安全數據。

# Monetization Hypothesis
- 提供免費的基本查詢服務，並對高級功能（如詳細報告和評價系統）收取訂閱費。
- 與承包商合作，提供廣告或推廣服務以增加收入。

# Success Metrics
- 每月活躍用戶數（MAU）。
- 用戶查詢次數。
- 承包商的註冊數量。
- 用戶滿意度調查結果。

# Risks & Assumptions
- **風險**：數據整合的準確性和及時性可能影響用戶信任。
- **假設**：用戶對承包商的安全記錄有足夠的關注，願意使用此平台進行查詢。
