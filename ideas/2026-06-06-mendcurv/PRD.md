# 產品名稱
Mendcurv

# One-line Pitch
一款幫助術後病人追蹤恢復進度的移動應用，並提供基於相似病例的恢復曲線參考。

# Background & Problem Statement
術後恢復過程中，病人常常面臨不確定性，尤其是對於疼痛程度、活動範圍及功能性里程碑的評估。病人需要一個明確的指標來判斷自己的恢復狀況是否正常。傳統的醫療溝通方式無法提供即時的數據支持，導致病人焦慮並頻繁聯繫醫療機構。Mendcurv 將通過收集病人的恢復數據，並與其他相似病人的恢復曲線進行比較，幫助病人更好地了解自己的恢復進度。

# Target Users & Personas
1. **術後病人**：如膝關節置換手術病人，渴望獲得即時的恢復進度反饋。
2. **家屬**：關心病人恢復狀況，想要了解病人的疼痛和活動情況。
3. **醫療專業人員**：希望能快速獲得病人恢復的數據，以便進行後續的醫療建議。

# User Pain Points
- 病人對於術後恢復的正常範圍缺乏了解。
- 傳統的醫療溝通無法即時解答病人的疑慮。
- 病人無法獲得與其他相似病例的比較數據，導致焦慮。

# Value Proposition
Mendcurv 提供一個簡單易用的界面，讓病人能夠每日記錄疼痛程度、活動範圍及功能性里程碑，並即時獲得與其他相似病人的恢復數據比較，讓病人清楚了解自己的恢復進度，減少焦慮感。

# Core Use Cases & User Stories
1. **記錄疼痛程度**：病人每天記錄自己的疼痛程度，系統自動生成恢復曲線。
   - *作為一名術後病人，我希望能夠方便地記錄我的疼痛程度，以便追蹤我的恢復進度。*
  
2. **查看恢復曲線**：病人可以查看與其他病人的恢復比較數據。
   - *作為一名術後病人，我希望能夠看到我的恢復進度與其他病人的比較，了解我的恢復是否正常。*

3. **獲取醫療建議**：根據病人的數據，醫療專業人員可以提供個性化建議。
   - *作為一名醫療專業人員，我希望能夠根據病人的數據提供針對性的建議，幫助病人更好地恢復。*

# MVP Scope (Must-have vs Nice-to-have)
## Must-have
- 用戶登錄/註冊系統
- 疼痛程度記錄功能
- 活動範圍記錄功能
- 功能性里程碑記錄功能
- 數據可視化界面
- 與其他病人恢復曲線的比較功能

## Nice-to-have
- 社區討論區
- 醫療專業人員的即時諮詢功能
- 數據導出功能（如 PDF 報告）

# User Flow Overview
1. 用戶註冊/登錄
2. 記錄疼痛程度、活動範圍及功能性里程碑
3. 查看個人數據及恢復曲線
4. 獲取與其他病人的比較數據
5. 接收醫療建議（如有）

# Basic System Architecture
- **前端**：React Native (移動應用)
- **後端**：Node.js + Express
- **數據庫**：MongoDB
- **數據可視化**：Chart.js
- **用戶認證**：JWT (JSON Web Token)

# Monetization Hypothesis
- **訂閱制**：提供免費基礎功能，收取月費或年費以解鎖進階功能（如專業醫療諮詢）。
- **合作醫療機構**：與醫療機構合作，提供數據分析服務以獲取收入。

# Success Metrics
- 用戶增長率
- 日活躍用戶數 (DAU)
- 用戶留存率
- 用戶滿意度調查結果

# Risks & Assumptions
- **風險**：用戶對數據隱私的擔憂，可能影響用戶採用。
- **假設**：病人會積極使用應用程序進行數據記錄，並願意分享其數據以獲得比較結果。
