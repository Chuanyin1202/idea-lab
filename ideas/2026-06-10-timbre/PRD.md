# 產品名稱
Timbre

## One-line Pitch
一款即時語音分析應用，幫助語言治療師更有效地評估和追蹤患者的語音進展。

## Background & Problem Statement
目前的語音治療診所依賴於數十年前的桌面分析軟體，這些系統經常崩潰，導致治療過程中斷。手動評分每位患者需耗時20分鐘，患者在每次約診後需等待數週才能進行下一次檢查，且在家練習的進展無法被量化。語言治療師需要可用的語音數據，但現有工具的工作流程卻消耗了可計費的時間，讓所有人都感到沮喪。

## Target Users & Personas
- **語言治療師**：需要高效的工具來評估和追蹤患者的語音進展。
- **患者**：希望能夠隨時隨地記錄自己的語音進展，並獲得即時反饋。

## User Pain Points
- 現有桌面軟體不穩定，經常崩潰。
- 手動評分耗時，影響治療效率。
- 患者在家練習的進展無法被量化，缺乏持續的動力。

## Value Proposition
Timbre 提供一個即時的語音分析工具，能夠快速生成治療報告，幫助語言治療師和患者之間建立更有效的互動。患者可以在家隨時記錄語音，並追蹤自己的進展，提升參與感和動力。

## Core Use Cases & User Stories
1. **治療師使用**：
   - 作為一名語言治療師，我希望在患者離開診所之前就能獲得語音分析報告，以便立即調整治療計畫。
   
2. **患者使用**：
   - 作為一名患者，我希望能在家中輕鬆記錄我的語音，並查看我的進展曲線，以保持我的學習動力。

## MVP Scope (Must-have vs Nice-to-have)
### Must-have
- 即時語音錄製功能
- 語音分析報告生成（包括音高穩定性、聲音粗糙度、共鳴等指標）
- 與治療師的報告同步功能

### Nice-to-have
- 患者進度追蹤儀表板
- 社群功能，讓患者之間可以分享進展
- 教學視頻和資源庫

## User Flow Overview
1. 患者打開應用，錄製60秒的語音樣本。
2. 系統生成語音分析報告。
3. 治療師在患者離開之前查看報告並提供反饋。
4. 患者在家中查看自己的進展曲線，並進行後續練習。

## Basic System Architecture
- **前端**：React Native (移動應用)
- **後端**：Node.js + Express (API)
- **數據庫**：MongoDB (存儲用戶數據和報告)
- **語音分析庫**：整合現有的開源語音分析庫
- **雲端服務**：AWS (存儲和計算)

## Monetization Hypothesis
每個診所每月收費299至499美元，根據使用量提供分級定價，確保收入與臨床採用緊密相連。

## Success Metrics
- 每月活躍用戶數 (MAU)
- 客戶留存率 (Retention Rate)
- 每位客戶的平均收入 (ARPU)
- 患者的進展滿意度調查結果

## Risks & Assumptions
- **風險**：語音分析的準確性未經驗證，可能影響治療師的信任。
- **假設**：語言治療師和患者都願意使用新的數位工具來改善治療效果。
