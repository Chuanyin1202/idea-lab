# 產品名稱
Lotvue

# One-line Pitch
幫助房主快速了解其地產是否能夠建造附屬住宅單位（ADU），節省時間和金錢。

# Background & Problem Statement
許多房主在考慮建造附屬住宅單位（ADU）時，往往需要花費數千美元來確認其地產是否符合建設要求。傳統上，建築師會收取約5000美元的可行性意見費用，而市政區域的文件通常複雜且難以理解，導致房主在研究過程中浪費數周的時間，最終得到的答案往往是模糊的。Lotvue旨在簡化這一過程，讓房主能夠在幾分鐘內獲得針對其具體地址的ADU可行性報告。

# Target Users & Personas
1. **房主**：希望利用自家地產增值，考慮建造ADU以出租或供家人居住。
2. **投資者**：尋找可增值的地產，並希望了解ADU的潛在收益。
3. **建築師/承包商**：需要快速獲取客戶地產的可行性信息，以便提供專業建議。

# User Pain Points
- 高昂的諮詢費用與時間成本。
- 難以理解的市政規範與建設要求。
- 缺乏針對特定地址的即時可行性評估工具。
- 對於建設過程的模糊性與不確定性。

# Value Proposition
Lotvue提供一個簡單易用的平台，房主只需輸入地址，即可獲得針對該地產的ADU可行性報告，報告中包含建設的可行性、類型、大小限制、預估建設成本及潛在租金收入，幫助房主做出明智的投資決策。

# Core Use Cases & User Stories
1. **房主使用案例**：
   - 作為一名房主，我希望能夠快速獲得關於我地產的ADU可行性報告，以便決定是否進行建設。
   
2. **投資者使用案例**：
   - 作為一名投資者，我希望能夠評估多個地產的ADU建設潛力，以便做出最佳投資選擇。

3. **建築師使用案例**：
   - 作為一名建築師，我希望能快速獲取客戶地產的可行性信息，以便提供準確的專業建議。

# MVP Scope (Must-have vs Nice-to-have)
## Must-have
- 地址輸入功能。
- 自動生成ADU可行性報告。
- 數據來源整合（地產資料、區域規範等）。
- 基本的用戶界面設計。

## Nice-to-have
- 移動應用版本。
- 用戶評價與反饋系統。
- 整合建築承包商的推薦系統。
- 高級報告功能（如詳細的市場分析）。

# User Flow Overview
1. 用戶訪問Lotvue平台。
2. 用戶輸入其地產地址。
3. 系統自動檢索相關數據。
4. 系統生成ADU可行性報告並展示給用戶。
5. 用戶可以下載報告或分享給其他人。

# Basic System Architecture
- **前端**：React.js或Vue.js
- **後端**：Node.js + Express
- **數據庫**：MongoDB或PostgreSQL
- **數據來源API**：整合地產數據API（如Zillow API）及市政規範API。
- **雲服務**：AWS或Google Cloud進行數據儲存與計算。

# Monetization Hypothesis
- 收取每份ADU可行性報告的費用（例如：$49）。
- 提供訂閱服務，讓用戶獲得持續的市場數據更新與分析。
- 與建築承包商合作，收取推薦費用。

# Success Metrics
- 每月活躍用戶數量。
- 每月生成的可行性報告數量。
- 用戶滿意度（通過調查收集反饋）。
- 收入增長率。

# Risks & Assumptions
- **風險**：數據來源的準確性與更新頻率可能影響報告的可靠性。
- **假設**：房主對於ADU的需求將持續增長，並且願意為快速獲得信息支付費用。
