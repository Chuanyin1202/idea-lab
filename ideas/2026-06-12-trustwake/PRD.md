# 產品名稱
Trustwake

# One-line Pitch
Trustwake 是一個專為遺產規劃公司設計的後續執行平台，幫助客戶完成信託資產的轉移與管理。

# Background & Problem Statement
許多遺產規劃公司在簽署信託合約後，無法有效追蹤客戶的資產轉移進度，導致高達60%的簽署信託未能資金化。客戶在缺乏清晰指示的情況下，往往會拖延進行必要的資產轉移，這不僅影響客戶的利益，也使得公司面臨法律責任。Trustwake 將提供一個系統化的解決方案，幫助公司追蹤客戶的進度並確保信託的有效執行。

# Target Users & Personas
- **遺產規劃顧問**：需要有效管理客戶信託的專業人士。
- **客戶**：簽署信託的個人，需完成資產轉移及更新受益人資訊。
- **財務顧問**：協助客戶進行資產管理的專業人士。

# User Pain Points
- 缺乏清晰的資產轉移指示，導致客戶拖延。
- 難以追蹤哪些信託文件已完成，哪些仍待處理。
- 客戶與顧問之間的溝通不暢，影響執行效率。
- 法律責任風險，尤其是在客戶未能資金化信託的情況下。

# Value Proposition
Trustwake 提供一個集中化的平台，幫助遺產規劃公司有效管理客戶的信託資產轉移。透過自動化的任務清單與提醒功能，顧問可以輕鬆追蹤客戶的進度，並降低法律風險，確保信託的有效執行。

# Core Use Cases & User Stories
1. **上傳信託文件**：遺產規劃顧問上傳簽署的信託文件，系統自動生成資產轉移任務清單。
2. **任務提醒**：系統定期向客戶發送資產轉移的提醒，確保客戶按時完成。
3. **進度追蹤**：顧問可以在儀表板上查看每位客戶的進度，及時跟進未處理的任務。
4. **完成證明上傳**：客戶在完成每項任務後，可以上傳證明文件，顧問可隨時查閱。

# MVP Scope (Must-have vs Nice-to-have)
## Must-have
- 文件上傳功能
- 自動生成任務清單
- 客戶提醒系統
- 進度儀表板

## Nice-to-have
- 整合財務顧問的聯絡資訊
- 提供法律風險評估報告
- 多語言支持

# User Flow Overview
1. 顧問上傳信託文件至平台。
2. 系統生成資產轉移任務清單並通知客戶。
3. 客戶按時完成任務並上傳證明。
4. 顧問在儀表板上檢視客戶的進度並進行後續跟進。

# Basic System Architecture
- **前端**：React.js
- **後端**：Node.js + Express
- **數據庫**：MongoDB
- **雲存儲**：AWS S3
- **身份驗證**：OAuth 2.0

# Monetization Hypothesis
Trustwake 將採取訂閱制收費模式，針對遺產規劃公司每月收取固定費用，並提供不同的訂閱層級以滿足不同需求。

# Success Metrics
- 客戶完成信託資產轉移的比例
- 使用平台的遺產規劃公司數量
- 客戶滿意度調查結果
- 平台的日活躍用戶數

# Risks & Assumptions
- **風險**：市場接受度不如預期，客戶對新系統的抵觸情緒。
- **假設**：遺產規劃公司願意投入資源以提升信託執行效率，並接受新的技術解決方案。
