# 產品名稱
Pepcheck

# One-line Pitch
一個專為購買複合型GLP-1藥物的患者設計的比較平台，提供即時價格、濃度及合法性資訊。

# Background & Problem Statement
隨著越來越多的患者選擇在線購買複合型藥物，市場上卻缺乏一個可靠的比較平台。患者在尋找合適的藥物時，常常面臨價格不一、合法性不明及供應商信譽不佳的問題。這導致患者在選擇時感到困惑，並可能購買到不合規或不安全的產品。

# Target Users & Personas
1. **患者**：尋找安全且價格合理的複合型GLP-1藥物的個人。
2. **醫療提供者**：希望為患者提供可靠的藥物資訊及建議的醫療專業人員。
3. **家屬**：幫助家人尋找合適藥物的照顧者。

# User Pain Points
- 缺乏透明的價格比較，導致患者無法做出明智的選擇。
- 不同地區的合規性資訊複雜，患者難以判斷供應商是否合法。
- 網上資訊來源不可靠，難以獲得真實的用戶評價。

# Value Proposition
Pepcheck 提供一個集中平台，讓患者能夠輕鬆比較不同供應商的價格、藥物濃度及合法性，並查閱經過驗證的用戶評價，從而做出明智的購買決策。

# Core Use Cases & User Stories
1. **用戶註冊**：患者可以創建帳戶以保存個人偏好及歷史記錄。
2. **價格比較**：用戶選擇地區後，平台顯示不同供應商的即時價格及藥物濃度。
3. **合法性查詢**：用戶可以查詢供應商的合法性及合規性資訊。
4. **用戶評價**：用戶可以查看其他患者的評價，並分享自己的購買經驗。

# MVP Scope
## Must-have
- 用戶註冊和登入功能
- 價格比較功能
- 合法性資訊查詢
- 用戶評價系統

## Nice-to-have
- 藥物濃度自動更新
- 提供個性化推薦
- 數據分析功能，幫助用戶了解市場趨勢

# User Flow Overview
1. 用戶註冊/登入
2. 選擇地區
3. 瀏覽並比較不同供應商的價格及合法性
4. 閱讀用戶評價
5. 完成購買或保存資訊以便日後參考

# Basic System Architecture
- **前端**：React.js
- **後端**：Node.js + Express
- **數據庫**：MongoDB
- **第三方API**：整合供應商的價格及合法性資料
- **雲服務**：AWS或Google Cloud進行數據存儲和服務部署

# Monetization Hypothesis
- 收取供應商的廣告費用以提升其在平台上的曝光率。
- 提供高級會員服務，讓用戶獲得更多的數據分析和個性化推薦。

# Success Metrics
- 每月活躍用戶數 (MAU)
- 用戶留存率
- 平台上完成的交易數量
- 用戶滿意度調查結果

# Risks & Assumptions
- 供應商的合法性資訊可能隨時變更，需持續更新數據。
- 市場競爭激烈，需確保平台的獨特性以吸引用戶。
- 用戶對於在線購買藥物的信任度可能影響平台的使用率。
