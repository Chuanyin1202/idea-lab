# 產品名稱
Codewire

# One-line Pitch
一個整合電氣計算的應用，幫助電工在現場快速獲取合規計算結果，避免錯誤與延誤。

# Background & Problem Statement
目前，電工在每個工作中必須使用五個不同的應用來進行計算，這些應用各自獨立，並且在每三年更新的電氣規範中，這些工具的更新時間不一，導致計算錯誤，可能會影響檢查結果或造成安全隱患。因此，電工需要一個集成的解決方案，能夠即時獲取最新的電氣規範和計算結果。

# Target Users & Personas
- **專業電工**：需要快速、準確的計算工具以確保合規性和安全性。
- **學徒電工**：需要學習和理解電氣計算，並希望能夠使用一個簡單的工具來輔助學習。
- **電氣工程師**：需要快速查詢和計算電氣設計的參數，並確保符合最新的規範。

# User Pain Points
- 多個應用之間的切換造成時間浪費。
- 每個應用的更新不一致，導致合規性風險。
- 缺乏一個集中式的資料庫來存儲和查詢計算歷史。
- 傳統的紙本參考資料不便攜帶且難以更新。

# Value Proposition
Codewire 提供一個整合的電氣計算平台，將所有必要的計算工具集中在一個應用中，並即時更新電氣規範，幫助電工提高工作效率，減少錯誤，並確保合規性。

# Core Use Cases & User Stories
1. **電工計算**：作為一名電工，我希望能夠在一個應用中進行所有必要的電氣計算，以節省時間並減少錯誤。
2. **合規查詢**：作為一名電工，我希望能夠隨時查詢最新的電氣規範，以確保我的工作符合標準。
3. **歷史記錄管理**：作為一名電工，我希望能夠保存我的計算歷史，以便未來查詢和參考。

# MVP Scope (Must-have vs Nice-to-have)
## Must-have
- 整合電壓降、導管填充、導管彎曲、箱子填充和電線安培容量計算器。
- 實時更新電氣規範資料庫。
- 計算結果的歷史記錄保存功能。

## Nice-to-have
- 材料清單生成器，將計算結果轉換為可購買的部件。
- 使用者界面自定義選項。
- 社群功能，讓用戶分享計算結果和經驗。

# User Flow Overview
1. 用戶登錄應用。
2. 選擇所需的計算工具。
3. 輸入相關數據並獲取計算結果。
4. 查詢最新的電氣規範。
5. 保存計算結果至歷史記錄。
6. 生成材料清單（若已解鎖此功能）。

# Basic System Architecture
- **前端**：React Native (移動應用)
- **後端**：Node.js + Express (API)
- **數據庫**：MongoDB (存儲計算歷史和用戶數據)
- **第三方服務**：使用 AWS Lambda 進行實時數據更新。

# Monetization Hypothesis
- 提供免費的基本計算工具，並通過訂閱模式解鎖進階功能如實時規範更新和材料清單生成。
- 可能的廣告收入來自於相關電氣產品的推廣。

# Success Metrics
- 用戶增長率：每月新增用戶數。
- 用戶留存率：每月活躍用戶比例。
- 計算使用頻率：每個用戶平均每天使用計算工具的次數。
- 訂閱轉換率：免費用戶轉換為付費用戶的比例。

# Risks & Assumptions
- 風險：市場競爭激烈，可能面臨其他類似應用的挑戰。
- 假設：用戶會接受並願意為集成的解決方案支付訂閱費用。
