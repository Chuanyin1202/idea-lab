# 產品名稱
Capax

# One-line Pitch
Capax 將員工的手機轉變為標準化的職業健康評估工具，快速提供回到工作崗位的清除認證。

# Background & Problem Statement
在許多企業中，員工因工傷需要進行回到工作崗位的清除認證，這通常需要通過診所預約進行評估，費用高達 300 到 600 美元，且預約時間長達數週。這導致員工在薪資上無法工作，企業因此承擔了加班費、臨時工費用和延長的傷殘支付。診所的可用性直接影響了整個流程的效率。

# Target Users & Personas
- **職業健康主管**：負責管理員工的健康狀況和回到工作崗位的流程，需快速獲得清除認證以減少成本。
- **自保企業**：管理多個工傷索賠的企業，需降低因延遲清除認證所造成的額外開支。
- **員工**：需要快速獲得回到工作崗位的認證，並希望能在家中進行評估。

# User Pain Points
- 長時間的診所預約導致員工無法工作。
- 高昂的評估費用增加了企業的負擔。
- 評估結果的延遲影響了企業的運營效率。
- 缺乏針對不同職位的具體評估標準。

# Value Proposition
Capax 提供一種便捷的解決方案，通過員工的手機進行標準化的健康評估，快速生成回到工作崗位的清除認證，從而節省時間和成本，並提高企業的運營效率。

# Core Use Cases & User Stories
1. **評估流程**：
   - 作為一名員工，我希望能夠使用手機進行健康評估，並在當天獲得結果，以便快速回到工作崗位。
   
2. **數據分析**：
   - 作為職業健康主管，我希望能夠查看所有員工的評估數據，以便做出更好的決策和改進健康管理流程。

3. **標準化評估**：
   - 作為一名自保企業的管理者，我希望能夠根據不同職位的需求設置評估標準，以確保評估的準確性和有效性。

# MVP Scope
- **Must-have**:
  - 手機評估功能（範圍運動測試、舉重能力檢查、平衡評估）。
  - 評估結果即時生成並發送給職業健康團隊。
  - 不同職位的功能基準庫。

- **Nice-to-have**:
  - 數據分析儀表板，供職業健康主管查看評估趨勢。
  - 整合與保險公司和第三方管理機構的合作功能。

# User Flow Overview
1. 員工下載並安裝 Capax 應用。
2. 員工註冊並選擇評估類型。
3. 員工根據指示進行手機評估。
4. 系統即時生成評估結果並發送給職業健康團隊。
5. 職業健康團隊根據結果做出回應。

# Basic System Architecture
- **前端**：React Native（移動應用）
- **後端**：Node.js + Express
- **數據庫**：MongoDB
- **雲服務**：AWS（用於存儲和計算）
- **視頻處理**：FFmpeg（用於視頻分析和處理）

# Monetization Hypothesis
- 單次評估收費 99 至 199 美元。
- 企業合約每月收費 2,500 至 5,000 美元，提供無限制的測試。

# Success Metrics
- 每月活躍用戶數（MAU）。
- 評估完成率。
- 客戶滿意度調查結果。
- 企業客戶的續約率。

# Risks & Assumptions
- 需要建立信任以獲得職業健康團隊的認可。
- 評估標準的準確性和有效性將直接影響產品的成功。
- 競爭者可能在未來進入市場，需持續優化產品以保持競爭力。
