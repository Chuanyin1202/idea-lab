# 產品名稱
Actuarium

# One-line Pitch
一款無偏見的長期護理保險計算器，幫助用戶比較自保與傳統保險選項的成本與收益。

# Background & Problem Statement
長期護理保險市場存在許多不透明的因素，保險經紀人通常會因為推薦特定的保險產品而獲得佣金，這使得消費者在選擇保險時難以獲得客觀的資訊。此外，許多家庭因為缺乏足夠的資訊而延遲做出保險決策，導致在需要時面臨高額的保險費用。Actuarium旨在提供一個透明且客觀的工具，幫助用戶在自保與購買保險之間做出明智的選擇。

# Target Users & Personas
- **主要用戶**: 58歲以上的成年人，擁有一定的資產（如$600K），需要考慮長期護理的需求。
- **次要用戶**: 家庭成員，負責照顧老年人或有長期護理需求的親屬。

# User Pain Points
1. 缺乏客觀的長期護理保險資訊，導致難以做出明智的決策。
2. 傳統保險產品的複雜性和不透明性使得消費者無法理解其實際成本。
3. 家庭成員在面對長期護理需求時感到無助和焦慮。

# Value Proposition
Actuarium提供一個簡單易用的計算器，透過客觀的數據分析，使用戶能夠清楚地比較自保與購買保險的成本與收益，從而做出更明智的財務決策。

# Core Use Cases & User Stories
1. **用戶故事**: 作為一名58歲的用戶，我希望輸入我的健康狀況和資產，並獲得不同保險選項的比較，以便我能做出明智的選擇。
2. **用戶故事**: 作為一名家庭成員，我希望能夠使用這個工具來了解長期護理的成本，從而幫助我的父母做出決策。

# MVP Scope (Must-have vs Nice-to-have)
- **Must-have**:
  - 用戶輸入健康狀況、資產和地區的功能。
  - 基於數據的長期護理保險選項比較。
  - 清晰的結果展示，包括成本和潛在收益的矩陣。

- **Nice-to-have**:
  - 用戶註冊和保存歷史計算結果的功能。
  - 提供專家建議或相關資源的連結。

# User Flow Overview
1. 用戶訪問Actuarium網站或應用程序。
2. 用戶輸入個人健康狀況、資產和地區。
3. 系統根據輸入的數據生成長期護理保險選項的比較。
4. 用戶查看結果並獲得建議。

# Basic System Architecture
- **前端**: React.js
- **後端**: Node.js + Express
- **數據庫**: MongoDB
- **數據來源**: 政府數據庫和精算表格API
- **雲服務**: AWS或Google Cloud

# Monetization Hypothesis
Actuarium可以通過提供增值服務（如專家諮詢或高級數據分析）來實現盈利，此外，也可以考慮與保險公司合作，提供推薦服務以獲取佣金。

# Success Metrics
1. 用戶註冊數量
2. 每月活躍用戶數
3. 用戶滿意度調查結果
4. 計算器使用次數

# Risks & Assumptions
- **風險**: 用戶對於輸入個人數據的隱私擔憂可能會影響使用率。
- **假設**: 用戶對於長期護理保險的需求會持續增長，並且願意使用數位工具進行比較。
