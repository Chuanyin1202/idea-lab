# 產品名稱
Unspin

# One-line Pitch
一款利用手機感測器指導用戶進行內耳治療的應用程式，提供便宜且有效的暈眩緩解方案。

# Background & Problem Statement
每月有368,000人搜尋「暈眩治療」，這顯示出市場對於解決暈眩問題的強烈需求。內耳結晶位移是最常見的暈眩原因，傳統的急診治療需要高達800美元的費用，且通常只會開藥物延遲症狀。物理治療則需要每次200美元的費用，且需多次就診。大多數人在家中嘗試自行治療時，因為缺乏精確的角度控制而失敗。

# Target Users & Personas
- **主要用戶**: 35-65歲的成年人，經常遭受暈眩困擾，尋求有效且經濟的解決方案。
- **次要用戶**: 醫療專業人員，尋求輔助工具以幫助患者進行自我治療。

# User Pain Points
- 高昂的醫療費用使得許多人無法負擔專業治療。
- 在家中自行治療時缺乏指導，容易導致錯誤的姿勢和治療失敗。
- 對於暈眩的長期影響缺乏有效的康復方案。

# Value Proposition
Unspin 提供一個低成本的解決方案，通過手機感測器實時指導用戶進行Epley操作，確保正確的姿勢和角度，從而有效緩解暈眩。用戶能夠在家中獲得專業的治療指導，並且後續提供康復進程的支持。

# Core Use Cases & User Stories
1. **用戶故事**: 作為一名經常感到暈眩的用戶，我希望能夠在家中使用Unspin應用程式，獲得準確的治療指導，從而減少就醫的需求。
2. **用戶故事**: 作為一名醫療專業人員，我希望能夠推薦Unspin給我的患者，幫助他們在家中進行有效的自我治療。

# MVP Scope (Must-have vs Nice-to-have)
- **Must-have**:
  - 手機感測器集成，能夠實時追蹤用戶的頭部角度。
  - Epley操作的語音指導。
  - 診斷檢查功能，識別受影響的耳朵和通道。
  
- **Nice-to-have**:
  - 用戶社區功能，分享經驗和支持。
  - 個性化康復計劃和進度追蹤。

# User Flow Overview
1. 用戶下載並安裝Unspin應用程式。
2. 用戶進行診斷檢查，確定受影響的耳朵。
3. 應用程式提供Epley操作的語音指導。
4. 用戶根據指導進行操作，應用程式實時追蹤角度並提供反饋。
5. 完成操作後，應用程式提供康復建議。

# Basic System Architecture
- **前端**: 使用React Native開發移動應用程式，支持iOS和Android。
- **後端**: Node.js和Express處理用戶請求，MongoDB存儲用戶數據。
- **感測器集成**: 使用手機內建的陀螺儀和加速度計進行數據收集。

# Monetization Hypothesis
- 透過訂閱模式，每月收取用戶費用，提供額外的康復內容和個性化服務。
- 提供免費試用期以吸引新用戶，之後轉為付費訂閱。

# Success Metrics
- 每月活躍用戶數 (MAU)。
- 用戶留存率。
- 用戶滿意度調查結果。
- 每月訂閱收入。

# Risks & Assumptions
- **風險**: 用戶可能對使用手機進行治療的安全性存有疑慮。
- **假設**: 用戶願意接受手機應用程式作為自我治療的工具，並能夠正確遵循指導。
