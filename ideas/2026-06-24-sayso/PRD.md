# 產品名稱
Sayso

## One-line Pitch
Sayso 是一款專為現場技術人員設計的語音助手，能夠將口述的工作內容自動轉換為結構化的工作訂單和庫存請求，提升工作效率。

## Background & Problem Statement
現場技術人員在執行任務時，經常需要在艱難的環境中（如水槽下、屋頂或爬行空間）進行工作，手部常常沾滿油污，無法方便地使用手機進行記錄。這導致了工作更新的延誤，並且常常出現記錄不完整的情況。現有的現場服務應用程式要求用戶手動輸入信息，這不僅耗時，還容易出錯。

## Target Users & Personas
- **技術人員**：在現場執行安裝、維修或檢查的專業人員，通常需要記錄工作進度和所需材料。
- **調度員**：負責管理技術人員的工作安排，需確保信息的準確性和及時性。
- **業主**：需要了解技術人員的工作進度和材料需求，以便做出相應的決策。

## User Pain Points
1. 難以在髒亂的環境中使用手機進行記錄。
2. 手動輸入信息耗時且容易出錯。
3. 工作更新不及時，導致調度員無法獲得準確的信息。
4. 需要在多個平台之間切換，增加了工作負擔。

## Value Proposition
Sayso 提供了一個語音驅動的解決方案，能夠簡化現場技術人員的工作流程。通過口述工作內容，系統能自動生成結構化的工作訂單和庫存請求，減少手動輸入的需求，提升工作效率，並確保信息的準確性。

## Core Use Cases & User Stories
1. **工作記錄**：技術人員在現場口述工作內容，Sayso 自動生成工作訂單。
   - *用戶故事*：作為一名技術人員，我希望能夠通過語音記錄我的工作，以便我可以專注於實際操作，而不是手動輸入信息。
   
2. **庫存請求**：技術人員口述所需的零件，Sayso 自動生成庫存請求。
   - *用戶故事*：作為一名技術人員，我希望能夠快速請求所需的零件，以免耽誤工作進度。

3. **拍攝照片**：技術人員可以通過語音指令拍攝照片並標記到工作訂單中。
   - *用戶故事*：作為一名技術人員，我希望能夠在不使用手的情況下拍攝照片，以便我可以專注於工作。

## MVP Scope (Must-have vs Nice-to-have)
### Must-have
- 語音識別功能，能夠將口述內容轉換為文本。
- 自動生成工作訂單和庫存請求的功能。
- 與現有現場服務平台（如 ServiceTitan、Jobber）的集成。

### Nice-to-have
- 照片拍攝和標記功能。
- 多語言支持。
- 數據分析和報告功能。

## User Flow Overview
1. 技術人員進入工作現場。
2. 使用 Sayso 語音助手開始記錄工作。
3. 語音助手將口述內容轉換為結構化數據。
4. 數據自動推送至相關的現場服務平台。
5. 技術人員可選擇拍攝照片並進行標記。

## Basic System Architecture
- **前端**：React Native（移動應用）
- **後端**：Node.js + Express
- **數據庫**：MongoDB
- **語音識別**：Google Cloud Speech-to-Text API
- **第三方集成**：ServiceTitan API, Jobber API

## Monetization Hypothesis
Sayso 將採用訂閱制收費模式，根據用戶數量和使用頻率收取月費或年費。此外，考慮提供增值服務，如數據分析報告，收取額外費用。

## Success Metrics
- 每月活躍用戶數（MAU）
- 用戶留存率
- 每位用戶的平均收入（ARPU）
- 工作訂單生成的準確性和及時性

## Risks & Assumptions
- 語音識別的準確性可能受到環境噪音的影響。
- 用戶對於新技術的接受度可能會影響產品的推廣。
- 需要與現有平台的集成可能會面臨技術挑戰。
