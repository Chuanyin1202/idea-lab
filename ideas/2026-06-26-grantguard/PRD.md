# 產品名稱
GrantGuard

# One-line Pitch
一個即時警示系統，幫助學生在調整課程時避免影響獎學金資格。

# Background & Problem Statement
社區大學的學生在調整學業計劃時，常常因為對獎學金資格的影響缺乏了解而面臨財務危機。獎學金的發放依賴於學分數、退課時間及學業進展等多重因素，這些因素的變化可能在學生提交課程調整請求後幾週才會顯現出來。這導致許多學生在未經過充分考慮的情況下做出決定，最終可能導致獎學金被撤回，甚至中斷學業。

# Target Users & Personas
- **學生**：正在就讀社區大學，面臨課程調整的需求，對獎學金規定不熟悉。
- **學校顧問**：負責指導學生的學業和財務決策，需提供準確的獎學金資訊。
- **財務辦公室人員**：需要即時了解學生的課程變化，並及時調整獎學金發放。

# User Pain Points
1. 學生在調整課程時缺乏對獎學金影響的即時了解。
2. 獎學金資格的變化往往在學生做出決策後才會顯現，造成意外的經濟壓力。
3. 學校顧問和財務辦公室之間的溝通不暢，導致信息傳遞延遲。

# Value Proposition
GrantGuard 提供即時的獎學金影響警示，幫助學生在調整課程時做出明智的決策，從而減少因獎學金資格變化而導致的經濟壓力，並促進學生的學業持續性。

# Core Use Cases & User Stories
1. **課程退選**：
   - 當學生嘗試退選一門課時，GrantGuard 會顯示該行為對獎學金的影響，並建議他們與顧問聯繫。
   
2. **轉為兼職學生**：
   - 學生在轉為兼職狀態時，系統會自動檢查其獎學金資格，並提供相應的建議。

3. **學業進展檢查**：
   - 當學生接近學業進展的邊界時，系統會提前警示，幫助他們調整學業計劃。

# MVP Scope
## Must-have
- 實時獎學金影響警示系統
- 課程調整的建議路由功能
- 學生資訊管理系統整合

## Nice-to-have
- 顧問和財務辦公室的溝通平台
- 學生獎學金歷史記錄查詢功能
- 數據分析報告生成器

# User Flow Overview
1. 學生登入系統。
2. 學生選擇要調整的課程。
3. 系統檢查獎學金影響並顯示警示。
4. 若有影響，系統提供顧問聯繫資訊。
5. 學生確認調整或聯繫顧問。

# Basic System Architecture
- **前端**：React.js
- **後端**：Node.js + Express
- **數據庫**：MongoDB
- **API整合**：RESTful API與學校的學生資訊系統整合
- **雲服務**：AWS或Azure進行數據存儲和應用部署

# Monetization Hypothesis
GrantGuard 可以通過向學校收取訂閱費用來實現盈利，並根據使用量提供不同的計費方案。此外，還可以考慮提供增值服務，如數據分析報告。

# Success Metrics
- 每月活躍用戶數（MAU）
- 學生獲得獎學金的成功率
- 用戶滿意度調查結果
- 系統使用頻率

# Risks & Assumptions
- 風險：學生對系統的依賴性可能導致其在獎學金管理上的主動性下降。
- 假設：學校會願意投入資源來整合此系統，並且學生會積極使用該系統來管理自己的學業和財務。
