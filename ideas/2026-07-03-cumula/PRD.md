# 產品名稱
Cumula

# One-line Pitch
Cumula 提供學校軟體堆疊的累積注意力負擔評分，幫助教育機構評估和管理數位工具對學生注意力的影響。

# Background & Problem Statement
隨著數位學習工具在學校的普及，教師和管理者面臨著一個未被充分認識的問題：不同應用程式對學生注意力的累積影響。每個應用程式可能看似無害，但當多個應用程式同時使用時，會造成學生的注意力負擔加重，影響學習效果。學校需要一個工具來評估這些應用程式的總體影響，並根據數據做出明智的採購決策。

# Target Users & Personas
- **學校管理者**：負責選擇和管理學校使用的數位工具，需向董事會報告應用程式的有效性。
- **教師**：使用各種應用程式輔助教學，需了解這些工具對學生的影響。
- **教育技術專家**：關注數位工具的有效性和對學生學習的影響。

# User Pain Points
- 無法量化各種應用程式對學生的注意力影響。
- 缺乏工具來評估和比較不同應用程式的數位健康程度。
- 學校管理者需要向董事會提供數據支持的報告，但缺乏相關數據。

# Value Proposition
Cumula 提供一個簡單易用的評分系統，幫助學校量化其數位工具的累積注意力負擔，並生成可視化報告，讓學校管理者能夠做出數據驅動的決策，提升學生的學習體驗。

# Core Use Cases & User Stories
1. **學校評估**：作為學校管理者，我希望能夠輸入學校使用的所有應用程式，並獲得一個總體的注意力負擔評分，以便向董事會報告。
2. **應用程式比較**：作為教師，我希望能夠比較不同應用程式的注意力影響，以選擇最適合學生的工具。
3. **年度報告生成**：作為學校管理者，我希望能生成年度報告，顯示隨著時間推移應用程式的影響變化。

# MVP Scope (Must-have vs Nice-to-have)
## Must-have
- 應用程式輸入功能
- 注意力負擔評分計算
- 可視化報告生成
- 基於 Common Sense Media 的應用程式評分數據整合

## Nice-to-have
- 應用程式使用數據分析
- 學校間的比較功能
- 自定義報告模板

# User Flow Overview
1. 用戶註冊並登錄系統。
2. 用戶輸入學校使用的應用程式清單。
3. 系統自動計算每個應用程式的注意力負擔評分。
4. 用戶生成報告並查看可視化數據。

# Basic System Architecture
- **前端**：React.js
- **後端**：Node.js + Express
- **數據庫**：MongoDB
- **數據來源**：整合 Common Sense Media API
- **雲服務**：AWS 或 Google Cloud

# Monetization Hypothesis
- 訂閱制：學校按年支付訂閱費用，獲取報告和數據分析服務。
- 額外服務：提供定制化報告和諮詢服務，收取額外費用。

# Success Metrics
- 每月活躍用戶數 (MAU)
- 訂閱轉化率
- 用戶滿意度調查結果
- 報告生成次數

# Risks & Assumptions
- 風險：學校對數位健康的重視程度不足，可能影響市場需求。
- 假設：學校願意為數位健康評估工具支付訂閱費用，並將其視為提升學習效果的必要投資。
