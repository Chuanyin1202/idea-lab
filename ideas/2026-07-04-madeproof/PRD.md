# 產品名稱
Madeproof

# One-line Pitch
一個連結產品與其來源的合規性平台，確保「美國製造」的標籤不再是空話。

# Background & Problem Statement
隨著消費者對產品來源的關注增加，許多品牌在包裝上標示「美國製造」，但往往無法提供足夠的證據來支持這一聲明。供應商可能會在不通知的情況下更換來源地，導致合規性問題。美國聯邦貿易委員會（FTC）已開始加強對這些聲明的監管，品牌需要一個可靠的系統來追蹤和管理其產品的來源信息，以避免潛在的法律風險和財務損失。

# Target Users & Personas
- **合規性官員**: 負責確保公司遵循法律法規，避免法律風險。
- **供應鏈經理**: 需要追蹤產品的來源和組件，以確保合規性和質量。
- **市場營銷團隊**: 需要準確的產品信息來支持市場推廣活動。

# User Pain Points
- 難以追蹤和管理產品的來源信息。
- 缺乏有效的工具來驗證和記錄供應商的變更。
- 在面對FTC檢查時，無法迅速提供所需的證據。

# Value Proposition
Madeproof 提供一個集中管理的平臺，將產品與其材料清單、供應商聲明和組裝地點相連結。它能即時更新供應商變更，並在包裝印刷前提醒用戶哪些聲明缺乏支持，從而降低合規風險。

# Core Use Cases & User Stories
1. **合規性檢查**: 合規性官員可以使用平台檢查每個產品的合規性狀態，確保所有聲明都有支持文件。
   - *用戶故事*: 作為合規性官員，我希望能快速查看每個產品的合規性狀態，以便在FTC檢查時提供必要的證據。

2. **供應商管理**: 供應鏈經理能夠追蹤供應商的變更，並自動更新相關產品的合規性聲明。
   - *用戶故事*: 作為供應鏈經理，我希望在供應商更換時自動收到通知，以便及時更新產品信息。

3. **市場推廣支持**: 市場營銷團隊可以輕鬆獲取準確的產品來源信息，以支持其推廣活動。
   - *用戶故事*: 作為市場營銷人員，我希望能快速獲取產品的來源信息，以便在廣告中準確使用。

# MVP Scope (Must-have vs Nice-to-have)
## Must-have
- 產品與材料清單連結功能
- 供應商變更通知系統
- 合規性狀態顯示（紅黃綠標示）
- 用戶管理與權限控制

## Nice-to-have
- 數據分析儀表板
- 自動生成合規性報告
- 整合第三方供應商數據庫

# User Flow Overview
1. 用戶登錄平台。
2. 用戶上傳產品信息和材料清單。
3. 系統自動檢查合規性並顯示狀態。
4. 用戶接收供應商變更的通知。
5. 用戶生成合規性報告，準備接受檢查。

# Basic System Architecture
- **前端**: React.js
- **後端**: Node.js + Express
- **數據庫**: PostgreSQL
- **雲服務**: AWS (S3 用於文件存儲，Lambda 用於無伺服器計算)
- **身份驗證**: OAuth 2.0

# Monetization Hypothesis
- 訂閱模式：根據用戶數量和功能層級收取月費。
- 服務費用：為特定合規性報告和數據分析提供按需收費服務。

# Success Metrics
- 每月活躍用戶數（MAU）
- 用戶留存率
- 合規性報告生成次數
- 收入增長率

# Risks & Assumptions
- **風險**: 競爭對手可能會推出類似的解決方案，影響市場份額。
- **假設**: 用戶願意為合規性管理工具支付訂閱費用，並且市場對這類工具有持續需求。
