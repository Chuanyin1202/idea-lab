# 產品名稱
Tactly

# One-line Pitch
一個自動化的發票跟進助手，幫助小型B2B團隊輕鬆收回未收款項，保持良好的客戶關係。

# Background & Problem Statement
許多小型企業面臨著未收款項的困擾，特別是在發票跟進的過程中。創業者通常需要親自跟進每一張未付款的發票，但這樣的過程往往讓人感到尷尬和不安，導致許多應收款項無法及時收回。現有的全功能應收賬款平台通常針對大公司，對於小型企業來說，這樣的解決方案過於繁瑣和昂貴。因此，我們需要一個簡單而有效的工具來幫助創業者自動化這一過程。

# Target Users & Personas
目標用戶為小型B2B企業的創業者和財務負責人。這些用戶通常負責多項業務，時間有限，並且希望能夠輕鬆管理應收賬款而不影響與客戶的關係。

## Personas
1. **創業者小李**：一位小型設計公司的創始人，負責所有業務運營，包括財務。他希望能夠簡化發票跟進的流程，減少與客戶的尷尬互動。
2. **財務經理小張**：在一家小型科技公司工作，負責處理發票和收款。他希望能夠自動化發票跟進，讓自己能專注於更重要的財務分析工作。

# User Pain Points
- 跟進未付款發票時感到尷尬，難以保持良好的客戶關係。
- 缺乏有效的工具來自動化發票跟進過程。
- 現有的應收賬款解決方案過於複雜且昂貴，無法滿足小型企業的需求。

# Value Proposition
Tactly 提供一個簡單易用的自動化發票跟進工具，能夠根據發票的狀態自動生成跟進郵件，並根據時間推移調整語氣，從而幫助小型企業輕鬆收回未收款項，同時保持良好的客戶關係。

# Core Use Cases & User Stories
1. **自動生成跟進郵件**：
   - 當發票超過到期日時，自動生成一封跟進郵件，語氣隨著逾期時間的增加而變得更直接。
   - 使用者小李希望在發票到期後自動發送跟進郵件。

2. **跟進狀態追蹤**：
   - 使用者可以查看每張發票的跟進狀態，了解哪些發票需要進一步跟進。
   - 財務經理小張希望能夠輕鬆查詢發票的狀態。

# MVP Scope (Must-have vs Nice-to-have)
## Must-have
- 與QuickBooks或Xero的集成。
- 自動生成跟進郵件的功能。
- 發票狀態追蹤面板。

## Nice-to-have
- 自定義郵件模板。
- 分析報告功能，顯示未收款項的趨勢。
- 提供支付選項的功能。

# User Flow Overview
1. 用戶連接其QuickBooks或Xero帳戶。
2. 系統自動導入發票數據並顯示在儀表板上。
3. 當發票超過到期日，系統自動生成跟進郵件。
4. 用戶可以查看發票狀態並手動發送郵件（如果需要）。

# Basic System Architecture
- **前端**：React.js
- **後端**：Node.js + Express
- **數據庫**：MongoDB
- **集成**：使用QuickBooks和Xero的API進行數據同步。

# Monetization Hypothesis
Tactly 將採取訂閱制的商業模式，提供不同級別的訂閱計劃，根據用戶的需求和發票數量收取月費。

# Success Metrics
- 每月活躍用戶數 (MAU)
- 用戶留存率
- 每月收回的未收款項總額
- 用戶滿意度調查結果

# Risks & Assumptions
- **風險**：用戶可能對自動化跟進郵件的效果持懷疑態度，可能影響使用率。
- **假設**：小型企業願意為簡化的發票跟進流程支付訂閱費用。
