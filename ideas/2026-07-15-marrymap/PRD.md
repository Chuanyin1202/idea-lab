# 產品名稱
Marrymap

# One-line Pitch
一款智能婚禮規劃應用，幫助新人逐步管理婚禮籌備的200個決策，節省時間與金錢。

# Background & Problem Statement
許多新人在籌備婚禮時面臨大量的決策和任務，通常需要花費約5,000美元聘請婚禮策劃師。這不僅增加了成本，還可能導致壓力和焦慮。Marrymap的目標是提供一個全面的解決方案，幫助新人有效地管理婚禮籌備過程，從而減少成本和壓力。

# Target Users & Personas
- **新人**：即將結婚的年輕夫婦，通常在25-35歲之間，對婚禮策劃缺乏經驗。
- **預算敏感型用戶**：希望控制婚禮開支，避免超支的年輕夫婦。
- **忙碌的職業人士**：工作繁忙，沒有時間進行繁瑣的婚禮籌備工作。

# User Pain Points
1. 婚禮籌備過程中的決策過多，容易感到不知所措。
2. 難以比較不同供應商的報價和服務。
3. 預算控制困難，容易超支。
4. 缺乏有效的時間管理工具，無法按時完成任務。

# Value Proposition
Marrymap提供一個集中化的平台，通過自動化的任務管理和預算追蹤，幫助新人輕鬆管理婚禮籌備過程。用戶可以根據婚禮日期和場地生成個性化的計劃時間表，並在一個儀表板上查看所有進度。

# Core Use Cases & User Stories
1. **生成計劃時間表**：作為一名用戶，我希望能輸入我的婚禮日期和場地，並自動生成詳細的計劃時間表。
2. **供應商報價比較**：作為一名用戶，我希望能夠將不同供應商的報價並排顯示，以便做出明智的決策。
3. **預算追蹤**：作為一名用戶，我希望能夠連接我的銀行數據，實時追蹤婚禮開支，避免超支。
4. **任務管理**：作為一名用戶，我希望能在儀表板上查看我的任務進度，並獲得即將到來的任務提醒。

# MVP Scope (Must-have vs Nice-to-have)
## Must-have
- 婚禮日期和場地輸入功能
- 自動生成計劃時間表
- 供應商報價比較工具
- 預算追蹤功能
- 任務管理儀表板

## Nice-to-have
- 自定義供應商聯繫模板
- 社交分享功能，與朋友分享進度
- 提供婚禮靈感和建議的內容庫

# User Flow Overview
1. 用戶註冊並輸入婚禮日期和場地。
2. 系統生成計劃時間表並顯示在儀表板上。
3. 用戶可以添加供應商並比較報價。
4. 用戶連接銀行賬戶以追蹤預算。
5. 用戶查看任務進度並獲取提醒。

# Basic System Architecture
- **前端**：React Native (移動應用)
- **後端**：Node.js + Express
- **數據庫**：MongoDB
- **第三方API**：銀行數據連接API，郵件發送API
- **雲服務**：AWS或Google Cloud (用於存儲和計算)

# Monetization Hypothesis
- 訂閱制：提供免費基礎版和收費高級版，高級版提供更多功能（如自定義模板和專業建議）。
- 供應商推廣：與婚禮相關的供應商合作，收取推廣費用。

# Success Metrics
- 每月活躍用戶數 (MAU)
- 用戶留存率
- 每位用戶平均收入 (ARPU)
- 供應商報價比較使用率

# Risks & Assumptions
- **風險**：市場競爭激烈，可能面臨其他婚禮應用的挑戰。
- **假設**：用戶願意為高級功能支付訂閱費，且能夠信任應用提供的預算追蹤和報價比較功能。
