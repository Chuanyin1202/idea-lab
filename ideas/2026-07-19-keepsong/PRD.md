# 產品名稱
Keepsong

## One-line Pitch
Keepsong 是一款專為失智症患者及其家庭設計的應用程式，透過故事提示和聲音錄音幫助重建家庭記憶和情感聯結。

## Background & Problem Statement
隨著失智症患者的增多，現有的醫療管理應用程式如 CareZone 雖然能夠追蹤藥物和約會，但卻缺乏情感層面的支持。家庭成員常常無法有效地保存和分享失智症患者的故事和回憶，導致情感聯繫的缺失。Keepsong 旨在填補這一空白，通過收集聲音記錄和照片，幫助家庭重建與親人的情感聯結。

## Target Users & Personas
- **主要用戶**：失智症患者及其家庭成員（如子女或配偶）。
- **次要用戶**：醫療護理人員和社會工作者，他們需要了解患者的背景故事以提供更好的照護。

## User Pain Points
1. 失智症患者對過去的記憶逐漸模糊，家庭成員無法有效地與他們溝通。
2. 現有應用程式無法提供情感支持，僅專注於醫療管理。
3. 家庭成員難以保存和整理患者的故事和回憶。

## Value Proposition
Keepsong 提供一個情感豐富的數位平台，使失智症患者能夠以自己的聲音講述故事，並透過家庭照片和故事提示來重建記憶，從而增強家庭成員之間的情感聯繫。

## Core Use Cases & User Stories
1. **用戶故事 1**：作為一名失智症患者，我希望能夠每天聽到與我相關的照片和故事提示，幫助我回憶過去的美好時光。
2. **用戶故事 2**：作為一名家庭成員，我希望能夠錄製父母的故事並自動整理成時間線，讓我能夠隨時回顧和分享這些珍貴的回憶。

## MVP Scope (Must-have vs Nice-to-have)
### Must-have
- 用戶登錄和註冊功能
- 每日檢查功能（顯示日期、天氣及家庭照片）
- 故事提示功能（引導用戶講述故事）
- 聲音錄音和保存功能
- 自動整理功能（按年代和主題整理故事）

### Nice-to-have
- 社交分享功能（與其他家庭成員分享故事）
- 故事回顧功能（回顧過去的錄音和照片）
- 整合健康管理功能（如藥物提醒）

## User Flow Overview
1. 用戶註冊並登錄應用程式。
2. 每日檢查顯示當天的日期、天氣和家庭照片。
3. 用戶根據故事提示錄製聲音，並保存至個人資料庫。
4. 錄音自動整理並顯示在時間線上，便於回顧和分享。

## Basic System Architecture
- **前端**：React Native（移動應用程式）
- **後端**：Node.js + Express
- **數據庫**：MongoDB（儲存用戶資料和故事）
- **雲存儲**：AWS S3（儲存聲音錄音和照片）

## Monetization Hypothesis
Keepsong 可以採用訂閱制模式，提供基本免費版本和高級付費版本，高級版本提供更多功能如無限錄音存儲和進階故事整理功能。

## Success Metrics
- 每月活躍用戶數（MAU）
- 用戶錄製的故事數量
- 用戶留存率
- 用戶滿意度調查結果

## Risks & Assumptions
- **風險**：用戶對於數據隱私的擔憂可能影響使用意願。
- **假設**：用戶會對情感支持功能有需求，並願意為此支付額外費用。
